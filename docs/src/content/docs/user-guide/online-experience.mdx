我们提供了几个直接在浏览器里运行的小 RTL 工程。这样你无需额外配置，也可以试用符号导航、代码理解、精准补全和自动重构的功能。

初次加载需要等待编辑器右上角的圆圈指示变绿，然后即可从已经选中的代码位置开始操作。你可以修改 RTL、切换文件、打开诊断面板，必要时可以使用工具栏里的重置按钮恢复初始状态。

## 符号导航

在以往的 Verilog 项目项目开发中，如果想要知道一个变量的定义或者使用位置，通常需要 Ctrl+F 来全局查找字符串，而这往往会把相似的变量名、不同作用域的重名符号带入搜索结果，大大影响了开发体验。

而在 Vide 中，你可以使用定义跳转、引用搜索和符号大纲在模块、端口和寄存器之间快速定位，让开发者不用离开当前上下文也能追清 RTL 连接关系。

本例展示了定义跳转、引用搜索功能。

### 操作步骤

1. 加载示例后，`top.v` 里的 `u_and_gate` 已经被选中。
2. 在 `u_and_gate` 或 `and_gate` 上执行 `转到定义`，跳到 `and_gate.v`。
3. 回到 `top.v`，在 `gate_out` 上执行查找引用或查看高亮。

## 代码理解

在以往的开发体验中，要找到模块签名、端口连接信息往往需要在文件中穿梭切换。或者如果想对一个数字字面量进行进制换算，往往需要打开外部工具。这些中断操作很容易打断开发的思绪。

在 Vide 中，我们提供了编辑处的悬停信息和代码注解，实现了在一个窗口就能中实时查看模块、字面量与端口连接信息，减少窗口切换的负担，让开发者更专注于 RTL 设计本身。

本例展示了悬停（模块签名、实例信息）和代码注解（端口提示）功能。

### 操作步骤

1. 加载示例后，查看 `u_mux` 每个连接旁边的端口提示。
2. 把鼠标停在 `mux2` 或 `u_mux` 上，查看模块和实例信息。
3. 打开 `mux2` 定义，对照端口列表和实例连接。

## 精准补全

使用文本编辑器、不依赖补全进行开发十分考验开发者对语言语法、项目结构的熟练程度；而如果补全功能只提供了字符串匹配，补全项中很容易出现相近的符号，无法大幅提高开发者的开发效率。

而 Vide 的补全机制理解当前的代码上下文，能在实例化、端口连接和其他的编辑位置给出更贴近工程语义的补全建议，更能通过代码片段提供结构化补全，达到事半功倍的效果。

### 操作步骤

1. 加载示例后，把光标移到 `adder8 u0(...)` 的端口列表里。
2. 删除一处端口名，或在新行输入 `.` 后触发补全。
3. 选择端口候选后，再在括号里补齐要连接的信号。

## 自动重构

开发者在重构代码时常常会有重复的工作，比如逐个补齐模块实例化时缺少的端口连接，把有序端口连接转换为命名连接，或者对一些符号进行重命名。这些重复性的工作本可以交给程序准确地完成。

Vide 提供了自动重构和重命名功能，把上述繁琐的细节交给 Vide 完成，解放开发者的重构体验。

本例展示了缺少端口连接的自动补齐功能和重命名功能。

### 操作步骤

1. 加载示例后，先手动删掉 `u_counter` 里的一行端口连接。
2. 把光标放回 `u_counter` 实例附近，打开代码操作菜单，执行端口补齐。
3. 在 `smp_cnt` 上执行重命名，检查声明、连接和使用点是否一起更新。
