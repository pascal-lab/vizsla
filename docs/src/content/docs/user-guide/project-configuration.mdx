这是一页 `vide.toml` 项目配置参考。第一次配置工程时，先按 [配置第一个项目](../first-project/) 写出能工作的项目配置文件。

## 字段速查

一个常见小工程会用到这些字段：

```toml
sources = ["rtl/**"]
include_dirs = ["include"]
defines = ["SYNTHESIS"]
top_modules = ["top"]
```

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `sources` | glob 数组或省略 | 选择工程源文件。常用写法是 `["rtl/**"]`。 |
| `include_dirs` | 路径数组或省略 | include 搜索目录，例如 `["include"]`。 |
| `defines` | 字符串数组 | 预定义宏。支持 `"NAME"` 和 `"NAME=value"`。 |
| `top_modules` | 字符串数组 | 声明当前工程的顶层模块名。 |
| `libraries` | 路径数组 | 外部库或依赖工程目录。 |
| `exclude` | glob 数组 | 从已加载文件中过滤生成目录、仿真输出或黑盒文件。 |

## `sources` 和 `exclude` 的路径和 glob

`sources` 和 `exclude` 不是普通目录路径，而是“带通配符的路径写法”，也叫 glob。最常用的两个通配符是：

- `*`：匹配一层目录里的名字，例如 `rtl/*.sv` 只匹配 `rtl` 目录正下方的 `.sv` 文件。
- `**`：匹配多层目录，例如 `rtl/**` 会继续匹配 `rtl` 子目录里的文件。

:::note
想了解这个概念，可以看 [glob 简介](https://en.wikipedia.org/wiki/Glob_(programming))。
:::

Vide 对 `sources` 和 `exclude` 有几条固定规则：

- 必须相对工作区根目录，例如 `rtl/**`。
- 路径分隔符统一写 `/`，不要写 `rtl\**`。
- 不支持绝对路径，也不支持路径里出现 `..`。
- `*` 不跨目录；需要递归目录时用 `**`。

在 `include_dirs` 和 `libraries` 里值是普通路径，并且相对当前项目配置文件所在目录解析。`include` 和 `include/` 表示同一个目录；为了统一，文档里推荐写成 `include`。

## 字段详解

### sources 和 include_dirs

`sources` 有三种常见状态：

| 写法 | 结果 |
| --- | --- |
| 省略 | Vide 会尽力索引工作区，用于跳转、引用、悬停等阅读功能，但不会把它当成完整工程配置。 |
| `sources = []` | 不自动扫描工作区。用于先打开工程，之后再手动补配置。 |
| `sources = ["rtl/**"]` | 按 glob 加载匹配文件，并让这些文件进入项目分析。 |

常用写法：

```toml
sources = ["rtl/**", "ip/**/*.sv", "tb/**/*.sv"]
```

当前工程加载器读取这些扩展名：

- `.v`
- `.sv`
- `.vh`
- `.svh`
- `.svi`
- `.map`

`include_dirs` 用于处理 `` `include "file.svh" ``。如果你已经写了 `sources`，但没有写 `include_dirs`，Vide 会从 `sources` 推导一个默认 include 目录。例如 `sources = ["rtl/**/*.sv"]` 会把 `rtl` 作为默认 include 目录。

如果 `sources` 被省略，Vide 不会从尽力索引结果反推 include 目录。显式写成 `include_dirs = []` 时，也不会使用这个回退。

### defines

`defines` 中每一项都是一个宏定义：

```toml
defines = [
  "SYNTHESIS",
  "WIDTH=32",
  "MODE=fast sim",
]
```

宏名需要是合法标识符。带值宏会按 `NAME=value` 解析，`value` 可以包含空格。

### libraries

`libraries` 用于声明依赖库。路径相对当前项目配置文件所在目录解析。每个库路径都会被当作另一个工作区继续加载：

- 如果库目录下有 `vide.toml`，Vide 会按这个项目配置文件加载库。
- 如果库目录没有项目配置文件，Vide 会把该目录作为未配置库加载。
- 依赖会参与当前工程的项目分析，并支持传递依赖。

`libraries` 不会替代当前工程自己的 `sources`。当前工作区的 RTL 仍然应该通过 `sources` 声明；`libraries` 只描述被当前工程引用的外部工程或库目录。

### exclude

`exclude` 使用工作区相对 glob，适合排除生成目录、仿真输出目录、缓存目录或黑盒 stub：

```toml
exclude = ["build/**", "out/**", "sim/work/**", "**/*_bb.v"]
```

`exclude = ["build"]` 只匹配名为 `build` 的路径本身，不会递归排除目录内容；目录递归请写 `build/**`。如果你还希望 VS Code 自己减少文件监听，可以同时配置 VS Code 的 `files.watcherExclude`。Vide 的 `exclude` 和 VS Code 的文件监听设置是两套机制。

## 常见注意事项

- 项目配置文件名必须是 `vide.toml`。
- Vide 不会从子目录自动发现项目配置文件。打开 `repo` 时，只检查 `repo/vide.toml`。


## 项目配置前后的行为差异

| 能力 | 没有项目配置文件 | 写好 `vide.toml` 后 |
| --- | --- | --- |
| [定义跳转](../features/navigation/)、[引用搜索](../features/references/) 与 [悬停信息](../features/hover/) | 扫描工作区，先提供读代码所需的跳转、引用和符号信息 | 按 `sources`、`libraries`、include 目录和宏分支解析真实目标 |
| [代码补全](../features/completion/) 与 [签名提示](../features/signature-help/) | 使用当前文件和已扫描文件给候选；目标模块解析后显示端口/参数提示 | 结合目标模块、include 目录和宏分支给出候选和当前位置提示 |
| [重命名](../features/rename/) 与 [自动重构](../features/quick-fixes/) | 局部代码操作可用；工程重命名不启用 | 按工程范围补齐/转换端口参数，并更新声明、引用和连接 |
| [语法高亮](../features/syntax-highlighting/) 与 [语义高亮](../features/semantic-highlighting/) | 语法高亮可用，当前文件有基础语义标记 | 结合项目解析标记端口方向、读写位置和宏分支 |
| [代码注解](../features/annotations/)、[符号大纲](../features/document-symbols/)、[折叠](../features/folding/) 与 [语义选区](../features/selection-range/) | 当前文件的大纲、折叠、选区可用；目标模块解析后显示代码注解 | 按工程结构显示端口、参数、实例数量、大纲、折叠和语义选区 |
| [实时诊断](../features/diagnostics/) | 打开的文件有语法和解析诊断；没有项目级语义诊断 | 使用 `sources`、`include_dirs`、`defines` 做跨文件语义诊断 |
| [骑河（Qihe）](../features/qihe/) | 使用当前文件作为分析输入 | 从 `sources`、`top_modules`、`include_dirs`、`defines` 推导编译参数 |
