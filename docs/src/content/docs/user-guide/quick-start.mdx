按下面的步骤，用几分钟确认 Vide 已经能在 VS Code 里工作。

## 1. 安装扩展

日常使用只需要安装 Marketplace 稳定版。安装后打开 RTL 工程目录，扩展会自动使用随扩展打包的 `vide` 语言服务器，不需要手动配置服务器路径。

也可以在 VS Code 的扩展面板搜索显示名 `Vide`，或搜索扩展 ID `vizsla.vizsla-lsp`。

![vide in extension market](../assets/quick-start/extension-marketplace.png)

## 2. 打开工程目录

用 VS Code 打开包含 RTL 源码的目录。后面文档里的「工作区根目录」，指的就是你在 VS Code 里打开的这个顶层目录。

如果这个目录里有 Verilog/SystemVerilog 源文件，但没有 `vide.toml`，扩展会提示你创建默认的项目配置文件。选择创建后，扩展会写入一个保守的 `vide.toml`，然后重新加载 Vide。

![创建项目配置文件提示](../assets/quick-start/create-manifest-notification.png)

默认配置会写入 `sources = []`。这表示 Vide 暂时只提供基础编辑体验，例如语法高亮、单文件错误提示、状态栏和命令；等工程目录确认清楚后再补 `sources`。

等你准备让 Vide 按真实工程理解文件关系时，再把源码目录写进 `sources`，例如：

```toml
sources = ["rtl/**"]
```

完整默认模板，以及 `sources = []` 和省略 `sources` 的区别，见 [第一个工程](../first-project/)。

## 3. 确认状态栏

扩展激活后，VS Code 右侧状态栏会出现名为 `Vide` 的状态项。正常启动后通常只显示 `Vide`；启动或停止时会显示旋转图标，缺少项目配置文件时会显示警告图标，启动失败或项目配置失败时会显示错误图标。把鼠标停在状态项上，可以看到当前详情。

![status bar](../assets/quick-start/status-bar.png)

点击状态项会打开状态菜单。中文界面里常用的入口包括：打开项目配置文件、创建项目配置文件、重新加载项目、重启语言服务器、分析诊断性能、显示输出。

![status menu](../assets/quick-start/status-menu.png)

## 4. 打开 Verilog/SystemVerilog 文件

打开 Verilog `.v`/`.vh` 文件，或 SystemVerilog `.sv`/`.svh`/`.svi` 文件。VS Code 应该识别对应语言并启用语法高亮和语言服务。

![example](../assets/quick-start/editor-example.png)

如果装好后没有反应，先确认你打开的是包含 `.v`、`.vh`、`.sv`、`.svh` 或 `.svi` 文件的目录，并打开了对应 RTL 文件。状态栏没有出现 `Vide` 时，可以重新加载 VS Code 窗口；状态栏显示错误或需要看输出时，先按上面的状态栏说明检查。需要替换服务器、安装本地 VSIX 或看启动日志时，再进入 [高级安装](../../advanced-guide/advanced-installation/) 和 [启动自检](../../advanced-guide/check-server/)。

## 5. 试用核心功能

你可以按这个顺序试一下：

1. 写一处明显语法错误，查看 VS Code `问题` 面板中的诊断，也就是错误、警告和提示。
2. 在模块名、信号名或实例名上执行 `转到定义` 或 `转到声明`，看看能不能跳到对应位置。
3. 在实例端口连接、参数赋值、表达式或预处理位置触发补全。
4. 把光标放到符号上查看悬停说明。
