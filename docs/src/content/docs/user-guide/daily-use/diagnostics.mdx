诊断会显示在 VS Code 的 `问题` 面板和编辑器下划线里。语法错误、解析错误通常只需要当前文件；跨文件语义诊断需要 `vide.toml` 里的 `sources`、`include_dirs`、`defines`、`libraries` 等项目信息。

诊断刷新和诊断规则属于 VS Code Settings。可以在设置界面搜索 `Vide Diagnostics`，也可以在用户或工作区 `settings.json` 里写入 `vide.diagnostics.*`；完整参考见 [Diagnostics](../../../advanced-guide/vscode-settings/#diagnostics)。

## 刷新策略

默认 `vide.diagnostics.update` 是 `onSave`，保存文件时刷新诊断。大型工程建议保持这个默认值，避免输入时反复触发较重分析。

需要输入时刷新时，在 VS Code 的用户或工作区 `settings.json` 中写：

```json
{
  "vide.diagnostics.update": "onType"
}
```

:::note[项目配置变化]
刚改过 `vide.toml` 时，执行 `Vide：重新加载项目配置` 会重新读取项目配置。只打开头文件，或文件未被 `sources` 匹配时，跨文件诊断通常不单独运行；配置方式见 [项目配置](../../project-configuration/)。
:::

## 调整诊断规则

把光标放在对应诊断位置，打开灯泡菜单。对于带有可识别诊断代码的 slang 诊断，Vide 会提供写入用户设置或工作区设置的快速修复，例如忽略此类诊断，或把错误降级为警告。

也可以在 VS Code Settings 里手动编辑 `vide.diagnostics.slang.rules`。规则写法见 [Diagnostics 设置参考](../../../advanced-guide/vscode-settings/#diagnostics)。
