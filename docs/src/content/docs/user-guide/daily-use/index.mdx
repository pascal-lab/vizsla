从日常任务入手：识别 RTL 文件、查看诊断、跳转阅读、补全与重构、格式化，以及从 VS Code 运行骑河（Qihe）。

配置分两处：

- 工程结构写在工作区的 `vide.toml`：源文件、include 目录、宏定义和库依赖。
- 编辑器行为放在 VS Code Settings：诊断刷新、行内提示、语义高亮、格式化、Qihe 命令等 `vide.*` 设置。按 `Ctrl+,` 搜索 `Vide`，或执行 `Preferences: Open User Settings (JSON)` / `Preferences: Open Workspace Settings (JSON)` 后写入对应键。

完整 `vide.*` 设置见 [VS Code 设置参考](../../advanced-guide/vscode-settings/)。
