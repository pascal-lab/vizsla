VS Code 扩展会注册两类语言：

| 语言 | 文件扩展名 |
| --- | --- |
| Verilog | `.v`, `.vh` |
| SystemVerilog | `.sv`, `.svh`, `.svi` |

打开这些文件后，VS Code 会先应用基础语法高亮、注释规则、括号匹配和文件图标。这部分不依赖后台语言服务器；即使服务器还在启动，基础编辑体验也应该可用。

语言服务器启动后，诊断、跳转、补全、悬停说明、签名帮助、行内提示、语义高亮、大纲、折叠、扩大/缩小选择和实例数量提示才会接入。需要跨文件理解时，工作区根目录需要有足够的项目配置。

相关设置参考：[Semantic Tokens](../../../advanced-guide/vscode-settings/#semantic-tokens)、[Files](../../../advanced-guide/vscode-settings/#files)。

## 语义高亮能多看出什么

语法高亮只知道关键字、数字和注释这类词法信息；语义高亮会结合语言服务器的分析结果。主题支持 semantic tokens 时，Vide 会额外标出端口方向、时钟/复位端口以及符号读写位置。

这里有几个比较实用的小提示：

- `input` / `output` 端口可以用不同颜色区分信号流向。
- clk / rst 端口可以使用更醒目的颜色，扫模块接口时更容易先定位时序边界。
- 读写位置可以区分显示，排查某个信号在哪里被赋值、在哪里被消费时更直观。

## 常见问题

### 文件没有被识别

先看 VS Code 右下角的语言模式。如果 `.v`、`.vh` 没有显示 Verilog，或 `.sv`、`.svh`、`.svi` 没有显示 SystemVerilog，手动点语言模式切换一次，确认是不是其它扩展覆盖了文件关联。

如果语法高亮有，但跳转、诊断、补全没有，通常不是语言识别问题，而是语言服务器或项目配置还没准备好。先点击状态栏的 `Vide` 看项目状态；如果提示没有项目配置，可以按 [第一个工程](../../first-project/) 创建或补全 `vide.toml`。

### 打开头文件时功能较少

`.vh`、`.svh` 和 `.svi` 通常通过被 `.v` 或 `.sv` 文件 include 后参与分析。只单独打开头文件时，语法高亮和局部编辑功能应该可用，但完整跨文件诊断不一定会单独跑。把包含它的源文件写入 `sources`，并把头文件目录写入 `include_dirs`，体验会更接近真实编译。
