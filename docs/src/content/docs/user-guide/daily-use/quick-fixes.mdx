Vide 会在实例端口、参数、部分诊断和可重命名符号附近提供代码操作。把光标放到相关位置后，打开 VS Code 灯泡菜单即可查看当前可用的修复或重构。

快速修复主要分两类：一类根据诊断或实例结构补齐/改写端口参数，另一类是不依赖诊断的局部重构，例如整数 literal 进制转换。可重命名符号则走 VS Code 的 `重命名符号` 命令。

诊断快速修复规则属于 VS Code Settings。可以在设置界面搜索 `Vide Diagnostics`，也可以在用户或工作区 `settings.json` 里写入 `vide.diagnostics.*`；完整参考见 [Diagnostics](../../../advanced-guide/vscode-settings/#diagnostics)。

## 支持的操作

把光标放到对应实例、数字 literal 或诊断附近，打开灯泡菜单。Vide 目前支持这些常见修复和重写：

- 端口没有接全：使用 `补全连接`。
- 参数没有值：使用 `补全参数`。
- 端口混用了有序写法和命名写法：使用 `将有序端口连接转换为命名连接`，或用 `移除空端口连接` 清理多余空连接。
- 参数混用了有序写法和命名写法：使用 `将有序参数赋值转换为命名赋值`。
- 写了 `.port` 但缺少 `()`：使用 `添加显式空端口连接`。
- 实例完全没有端口列表：使用 `添加空实例端口列表`。
- 整数 literal 进制不合适：在数字附近使用 `将字面量转换为二进制`、`将字面量转换为八进制`、`将字面量转换为十进制` 或 `将字面量转换为十六进制`。

:::note[项目范围]
实例端口和参数相关操作需要目标模块能被解析到。跨文件项目里，`sources`、`include_dirs`、`defines` 和 `libraries` 决定 Vide 能看到哪些模块。
:::

## 重命名范围

重命名依赖符号解析结果。`查找所有引用` 和 `重命名符号` 使用同一个项目视图；补充项目配置后执行 `Vide：重新加载项目配置` 会刷新引用和重命名范围。
