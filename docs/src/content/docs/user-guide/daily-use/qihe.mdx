执行 `Vide：运行 Qihe 分析` 可以对当前本地 Verilog/SystemVerilog 文件运行 Qihe。命令只对 `file:` URI 可用，文件扩展名必须是 `.v`、`.vh`、`.sv`、`.svh` 或 `.svi`。

相关设置参考：[Qihe](../../../advanced-guide/vscode-settings/#qihe)。

## 如何运行

1. 在 VS Code 中打开要分析的 RTL 文件。
2. 点击编辑器标题栏里的 <Codicon name="beaker" label="运行 Qihe 分析图标" /> 烧杯图标，或从命令面板执行 `Vide：运行 Qihe 分析`。
3. 运行时会出现独立的 <Codicon name="beaker" label="Qihe 状态图标" /> `Qihe` 状态项。
4. 失败后点击这个状态项，或点击错误通知里的 `显示 Qihe 输出`，查看 `Vide Qihe` 输出通道。

## 参数从哪里来

默认情况下，Vide 会从当前 `vide.toml` 项目配置文件推导 Qihe 编译模式、顶层模块、include 目录和宏定义。

如果项目已经用脚本管理 Qihe 参数，可以关闭自动推导并显式配置：

```json
{
  "vide.qihe.autoConfigureArgsFromManifest": false,
  "vide.qihe.compileArgs": ["--mode", "sv", "--", "-I", "include"],
  "vide.qihe.runArgs": ["-g", "std"]
}
```

需要指定 Qihe 可执行文件时，配置 `vide.qihe.command`。Windows 上从桌面启动 VS Code 时看到的 `PATH` 可能和终端不同；不确定时写绝对路径。

## 命令不可用或按钮没有效果

先确认目标是本地文件，不是远程、虚拟文档或未保存文件。再确认扩展名属于 Verilog/SystemVerilog。命令仍不可用时，切换到实际 RTL 编辑器标签页后重试。

## 分析失败

打开 `Vide Qihe` 输出通道，按顺序看目标文件、编译参数、运行参数、Qihe stdout/stderr 和最后的失败信息。若失败与 include、宏或 top module 有关，先修正 [项目配置](../../project-configuration/)；若失败与 Qihe 命令路径有关，优先把 `vide.qihe.command` 改成绝对路径。

想了解为什么有时按工程分析、有时退回单文件分析，见 [Qihe 工程分析](../../../advanced-guide/parsing-and-analysis/#qihe-工程分析)。
