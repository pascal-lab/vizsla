执行 `Vide：运行 Qihe 分析` 可以对当前本地 Verilog/SystemVerilog 文件运行骑河（Qihe）分析。命令只对 `file:` URI 可用，文件扩展名必须是 `.v`、`.vh`、`.sv`、`.svh` 或 `.svi`。

Qihe 运行选项属于 VS Code Settings。可以在设置界面搜索 `Vide Qihe`，也可以在用户或工作区 `settings.json` 里写入 `vide.qihe.*`；完整参考见 [骑河（Qihe）](../../../advanced-guide/vscode-settings/#qihe)。

## 如何运行

1. 在 VS Code 中打开要分析的 RTL 文件。
2. 点击编辑器标题栏里的 <Codicon name="beaker" label="运行 Qihe 分析图标" /> 烧杯图标，或从命令面板执行 `Vide：运行 Qihe 分析`。
3. 运行时会出现独立的 <Codicon name="beaker" label="Qihe 状态图标" /> `Qihe` 状态项。
4. 运行后点击这个状态项，或点击通知里的 `显示 Qihe 输出`，查看 `Vide Qihe` 输出通道。

## 参数从哪里来

默认情况下，Vide 会从当前 `vide.toml` 项目配置文件推导骑河（Qihe）编译模式、顶层模块、include 目录和宏定义。

项目已经用脚本管理骑河（Qihe）参数时，可以关闭自动推导并显式配置：

```json
{
  "vide.qihe.autoConfigureArgsFromManifest": false,
  "vide.qihe.compileArgs": ["--mode", "sv", "--", "-I", "include"],
  "vide.qihe.runArgs": ["-g", "std"]
}
```

需要指定骑河（Qihe）可执行文件时，配置 `vide.qihe.command`。Windows 上从桌面启动 VS Code 时看到的 `PATH` 可能和终端不同；工作区配置里写绝对路径可以固定使用的 Qihe 版本。

## 运行范围

这个命令面向本地已保存的 RTL 文件：目标必须是 `file:` URI，扩展名必须是 `.v`、`.vh`、`.sv`、`.svh` 或 `.svi`。远程文件、虚拟文档和未保存缓冲区不会作为 Qihe 分析目标。

## 输出内容

`Vide Qihe` 输出通道会记录目标文件、编译参数、运行参数、Qihe stdout/stderr 和最后的结果信息。include 目录、宏定义和 top module 默认来自 [项目配置](../../project-configuration/)；骑河（Qihe）可执行文件由 `vide.qihe.command` 指定。

想了解为什么有时按工程分析、有时退回单文件分析，见 [骑河（Qihe）工程分析](../../../advanced-guide/parsing-and-analysis/#qihe-工程分析)。
