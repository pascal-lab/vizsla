格式化入口包括 VS Code 的 `格式化文档`、`格式化选定内容` 和按 Enter 时的自动缩进行为。当前格式化后端只有 `verible`，会调用外部 `verible-verilog-format`；Vide 不随扩展打包这个工具。

`vide.formatter.*` 和 `vide.formatting.*` 都是 VS Code Settings，不写在 `vide.toml`。可以在设置界面搜索 `Vide Formatter`，也可以在用户或工作区 `settings.json` 里写入这些键。完整参考见 [Formatter 和 Formatting](../../../advanced-guide/vscode-settings/#formatter-和-formatting)。

## 配置 verible

`vide.formatter.provider` 目前只有 `verible`。`vide.formatter.path` 留空时，Vide 会从 VS Code 进程可见的 `PATH` 查找 `verible-verilog-format`；写入路径时，会直接调用这个可执行文件：

```json
{
  "vide.formatter.path": "D:\\tools\\verible\\verible-verilog-format.exe"
}
```

## 格式化入口

`格式化文档` 会把整个文件交给 `verible-verilog-format`。`格式化选定内容` 会把选区对应的行范围传给同一个 formatter。两者都会使用当前的 `vide.formatter.path` 和 `vide.formatter.args`；`vide.formatting.indent.width` 会作为 `--indentation_spaces` 传入。

## Enter 行为

按 Enter 的辅助格式化由 `vide.formatting.on.enter` 控制，注释里的 Enter 行为由 `vide.formatting.in.comments` 控制。项目只需要手动格式化时，可以关闭 Enter 行为，保留 `格式化文档` 和 `格式化选定内容`。
