Vide 会在实例连接、参数赋值、表达式、预处理指令和常见 RTL 结构附近提供补全。补全结果依赖当前语法位置和已经解析到的项目视图。

常见触发场景包括：

- 在实例参数 `#(...)` 和端口 `(...)` 中补全目标参数、目标端口和连接表达式。
- 在表达式、块语句、generate/specify/config 等语法位置补全可用符号和关键字片段。
- 输入 `$` 时补全 SystemVerilog 系统任务/函数，输入 `` ` `` 时补全预处理指令。
- 输入 `'` 时补全整数 literal 的进制标记，例如 `b`、`o`、`d`、`h`。
- 在事件控制 `@(...)`、成员访问 `.` 等位置补全当前语义上下文里的候选项。

## 常见问题

### 补全内容太少

单文件场景下，补全通常只能利用当前文件和已索引到的内容。端口、参数、实例相关补全需要目标模块可解析；include、宏和库依赖也要在项目配置中写清楚。

如果实例模块名本身跳不到定义，先回到 [导航和阅读](../navigation/#跳不到定义) 排查项目配置。实例端口和参数列表里的调用提示见 [签名帮助](../signature-help/)。
