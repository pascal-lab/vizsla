## 什么是重命名

重命名会把同一个 RTL 符号的声明和引用一起更名。它比手动搜索替换更安全，因为 Vide 会先解析当前位置对应的符号，再生成 VS Code 工作区编辑。

## 项目配置影响

重命名依赖符号解析和可编辑的工程范围。没有 `vide.toml` 时，Vide 会用工作区扫描结果支持阅读类引用搜索，但不会启用工程重命名；把相关源文件写进 `sources` 后，执行 `Vide：重新加载项目配置` 会刷新重命名范围。

## 使用边界

如果当前位置不能解析为唯一符号，或者目标文件不在当前项目配置管理的可编辑范围内，VS Code 不会进入重命名流程。需要先确认 [定义跳转](../navigation/) 和 [引用搜索](../references/) 能定位到同一个目标。
