本页面面向已经使用 Neovim 内置 LSP 的用户。我们会教你下载 Vide 语言服务器，并配置 Neovim 在打开 Verilog/SystemVerilog 文件时启动它，把跳转、补全、悬停、诊断等能力接入编辑器界面。

## 1. 下载 Vide

从下面选择与你平台匹配的语言服务器二进制，并将它重命名为 `vide`（Windows 为 `vide.exe`）：

| 平台 | 下载链接 |
| --- | --- |
| Linux x64 | [`vide-linux-x64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-linux-x64) |
| Linux arm64 | [`vide-linux-arm64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-linux-arm64) |
| Windows x64 | [`vide-win32-x64.exe`](https://github.com/pascal-lab/vide/releases/latest/download/vide-win32-x64.exe) |
| macOS arm64 | [`vide-darwin-arm64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-darwin-arm64) |
| Alpine/musl x64 | [`vide-alpine-x64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-alpine-x64) |
| Alpine/musl arm64 | [`vide-alpine-arm64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-alpine-arm64) |

Linux 或 macOS 可以用以下命令（注意把 `<platform>` 替换为对应平台名称）：

```bash
mkdir -p ~/.local/bin
curl -L https://github.com/pascal-lab/vide/releases/latest/download/vide-<platform> -o ~/.local/bin/vide
chmod +x ~/.local/bin/vide
vide --version
```

Windows PowerShell 示例：

```powershell
New-Item -ItemType Directory -Force "$env:LOCALAPPDATA\vide"
Invoke-WebRequest "https://github.com/pascal-lab/vide/releases/latest/download/vide-win32-x64.exe" -OutFile "$env:LOCALAPPDATA\vide\vide.exe"
& "$env:LOCALAPPDATA\vide\vide.exe" --version
```

下载后，请确认 `vide --version` 可以执行并返回版本信息。

你可以将 `vide`（或 `vide.exe`）所在目录加入环境变量 `PATH` 中，以方便后续操作。当然，这不是必须的。

## 2. 找到 Neovim 配置文件

接下来我们要修改 Neovim 的配置。你可以在 Neovim 里执行下面两条命令，分别确认配置目录和当前配置文件的位置：

```vim
:echo stdpath('config')
:echo $MYVIMRC
```

如果第二条命令输出为空，说明你还没有现成的配置文件，可以在第一条命令输出的对应目录下新建 `init.lua`。比如 Windows 上执行第一条命令通常会看到类似输出：

```text
C:\Users\<user>\AppData\Local\nvim
```

对应的配置文件就新建到 `C:\Users\<user>\AppData\Local\nvim\init.lua`。

## 3. 配置 Neovim 启动 Vide

请将下面这段配置加入刚才提到的配置文件 `init.lua`：

```lua
vim.filetype.add({
  extension = {
    v = 'verilog',
    vh = 'verilog',
    sv = 'systemverilog',
    svh = 'systemverilog',
    svi = 'systemverilog',
  },
})

vim.lsp.config('vide', {
  cmd = { 'vide' },
  filetypes = { 'verilog', 'systemverilog' },
  root_markers = { 'vide.toml', '.git' },
})

vim.lsp.enable('vide')
```

这段配置做了两件事：

- 显式把 `.v`、`.vh`、`.sv`、`.svh` 和 `.svi` 识别为 `verilog` 或 `systemverilog`，避免不同 Neovim 版本或个人配置带来的默认识别差异。
- 告诉 Neovim 在这些文件类型里启动 `vide` 语言服务器，并用 `vide.toml` 或 `.git` 作为项目根目录标记。

如果第一步下载的 `vide` 不在 `PATH` 中，请把 `cmd` 改成绝对路径，例如：

```lua
cmd = { 'D:/vide/vide.exe' }
```

打开文件后，你也可以用下面的命令确认当前文件类型是否正确：

```vim
:set filetype?
```

期望结果是 `.v`/`.vh` 显示 `filetype=verilog`，`.sv`/`.svh`/`.svi` 显示 `filetype=systemverilog`。

## 4. 准备项目配置

在项目的根目录编写 `vide.toml`，让 Vide 知道要分析哪些源文件、头文件目录和宏定义。第一次配置可以先看[配置第一个项目](../first-project/)。

## 5. 验证

打开工程中的 `.v` 或 `.sv` 文件，然后在 Neovim 中执行：

```vim
:lua print(vim.inspect(vim.lsp.get_clients({ name = 'vide' })))
```

如果输出里能看到 `vide` client，说明语言服务器已经启动。也可以直接在代码中试试跳转定义、悬停、补全或诊断。
