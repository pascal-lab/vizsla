VS Code 设置控制编辑器里的行为，例如诊断刷新、Qihe 命令、格式化工具、行内提示和语言服务器启动参数。工程源文件、include 目录、宏定义和依赖库请写在 [项目配置](../project-configuration/) 的 `vide.toml` 里。

所有设置都在 `vide.*` 命名空间下。可以在 VS Code 设置界面搜索 `Vide`，也可以直接编辑用户或工作区的 `settings.json`。

## 常用设置速查

| 你想做什么 | 常用设置 |
| --- | --- |
| 让 VS Code 调用本机的 Qihe | `vide.qihe.command` |
| 指定本机的 `verible-verilog-format` | `vide.formatter.path` |
| 控制诊断刷新是保存后还是输入时 | `vide.diagnostics.update` |
| 开关端口、参数、结构结尾的行内提示 | `vide.inlayHints.*` |
| 开关模块声明上方的实例数量提示 | `vide.lens.instantiations.enable` |
| 项目配置变化后是否自动刷新 | `vide.workspace.auto.reload` |

服务器启动命令、文件监听、诊断规则和通信跟踪更偏排障或开发场景；常规使用保持默认值即可。

## Server

这组设置用于替换或调试后台语言服务器。常规使用保持默认值即可。

<VscodeSettingsTable
  locale="zh"
  settings={[
    'vide.server.command',
    'vide.server.args',
    'vide.server.additionalArgs',
    'vide.server.cwd',
    'vide.trace.server',
  ]}
/>

这些服务器启动设置变更后，扩展会提示 `重启`：`vide.server.command`、`vide.server.args`、`vide.server.additionalArgs`、`vide.server.cwd`、`vide.trace.server`。

## Qihe

运行 `Vide：运行 Qihe 分析` 时才需要看这组设置。默认情况下，Vide 会从当前 `vide.toml` 推导 Qihe 编译模式、顶层模块、include 目录和宏定义。

相关功能：[骑河分析](../features/qihe/)。

<VscodeSettingsTable
  locale="zh"
  settings={[
    'vide.qihe.command',
    'vide.qihe.autoConfigureArgsFromManifest',
    'vide.qihe.compileArgs',
    'vide.qihe.runArgs',
  ]}
/>

相关文档：[项目配置参考](../project-configuration/)。
