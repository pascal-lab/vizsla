这页用于查询 Vide 的用户配置。用户配置用来改变语言服务器的行为，例如诊断什么时候刷新、是否显示代码注解、使用哪个格式化工具、运行 Qihe 时带哪些参数。

用户配置不负责描述工程里有哪些文件。源文件、include 目录、宏定义、依赖库和顶层模块应该写在 `vide.toml` 里；第一次配置工程时请看 [配置第一个项目](../first-project/)。

## 写在哪里

同一个选项在不同编辑器里的写法不完全一样。

VS Code 用户可以在设置界面搜索 `Vide`，也可以直接编辑 `settings.json`。VS Code 的设置名带 `vide.` 前缀：

```json
{
  "vide.diagnostics.update": "onType",
  "vide.inlayHints.port.connection.enable": true
}
```

Neovim、Emacs 等编辑器通常通过 LSP 启动 Vide。LSP 是编辑器和语言服务器之间的通信协议；这类编辑器通常把同样的配置作为初始化选项传给 Vide。初始化选项就是编辑器启动语言服务器时一起发送的 JSON 对象。这个写法不带 `vide.` 前缀：

```json
{
  "diagnostics": {
    "update": "onType"
  },
  "inlayHints": {
    "port": {
      "connection": {
        "enable": true
      }
    }
  }
}
```

如果你使用 Neovim、Emacs 或其他 LSP 编辑器，进阶页里有完整的放置位置和示例：[LSP 编辑器的用户配置](../../advanced-guide/user-configuration/)。

## 如何读下面的表

表里的“用户配置”列使用点分路径表示 JSON 中的位置。例如 `diagnostics.update` 对应：

```json
{
  "diagnostics": {
    "update": "onSave"
  }
}
```

“VS Code 中的名字”列表示同一个选项在 VS Code `settings.json` 里的名字。例如 `diagnostics.update` 对应 `vide.diagnostics.update`。

建议从默认值开始，只在确实需要时再修改。

## 配置校验文件

JSON schema 是一份描述配置结构的文件。编辑器或配置工具支持 JSON schema 时，可以用它提供补全，并在启动服务器前发现拼错字段的问题，例如把 `diagnostics` 写成 `diagnostic`。

用户配置的 schema 地址是：

```text
https://vide.pascal-lab.net/schemas/v1/user-config.schema.json
```

如果你把用户配置保存成单独的 JSON 文件，可以在文件顶部加 `$schema` 指向这个地址。`$schema` 只帮助编辑器补全和校验，Vide 读取配置时会忽略它。直接写在 Neovim `init_options` 或 Emacs `:initializationOptions` 里时，通常不需要 `$schema`。
