## 什么是代码补全

代码补全是在当前位置给出可插入的候选项。Vide 的补全会结合语法位置和语义尽可能给出合适的、抑制不合适的补全项：在实例端口列表里补目标端口，在参数列表里补目标参数，在表达式里补当前可见符号，在预处理位置补指令和宏相关内容。

常见触发位置包括实例 `#(...)` 和 `(...)`、表达式、成员访问 `.`、事件控制 `@(...)`、系统任务 `$`、预处理反引号 `` ` `` 和整数 literal 进制标记 `'`。

## 项目配置影响

### 没有完整配置时

补全会使用当前文件和已经扫描到的内容。表达式里的局部符号、当前文件里的模块和基础语法位置通常可以补全；如果目标模块在别的文件里，端口和参数补全可能不完整。

### 有完整配置时

写好 `vide.toml` 后，Vide 会按工程范围解析模块、include 文件和宏分支。实例端口、参数赋值和跨文件符号补全会更准确。
