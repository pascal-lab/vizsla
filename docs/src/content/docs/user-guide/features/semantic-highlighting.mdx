## 什么是语义高亮

语法高亮只知道一个词长得像关键字、数字还是注释；语义高亮会结合 Vide 的解析结果，让同样是标识符的 RTL 名字呈现出不同样式。读代码时，你可以更快区分时钟/复位端口、输入输出端口、模块实例名和 `typedef` 类型；在默认显示里，输入端口会加粗（例如 <strong>data_i</strong>），输出端口会加粗并带下划线（例如 <strong><u>data_o</u></strong>），时钟和复位端口也会使用区别于普通端口的颜色。

## 项目配置影响

当前文件里的基础语义标记可以直接计算；跨文件解析、宏分支和 include 关系会影响目标模块、端口和声明的判定。写好 `vide.toml` 后，语义高亮会按 `vide.toml` 描述的工程结构刷新。

## 调整显示

语义高亮需要当前 VS Code 主题支持 semantic tokens。Vide 还提供两个开关：

- `vide.semantic.tokens.port.clk.rst.enable`：控制时钟/复位端口标记。
- `vide.semantic.tokens.port.input.output.enable`：控制 input/output 端口方向标记。

完整写法见 [Semantic Tokens 设置参考](../../vscode-settings/#semantic-tokens)。
