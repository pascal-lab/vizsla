## 什么是诊断

诊断就是 Vide 让编辑器显示的 <Codicon name="error" label="报错图标" color="#f14c4c" /> 报错、<Codicon name="warning" label="警告图标" color="#facc15" /> 警告和 <Codicon name="info" label="提示图标" color="#3794ff" /> 提示。它会出现在 VS Code 的 `问题` 面板、编辑器下划线和对应代码位置附近，帮助你发现语法错误、解析错误，以及项目分析发现的语义问题。

Vide 的常规语法/语义诊断主要来自 slang，并会把这些诊断转换成编辑器里的标记和问题列表。需要运行更深入的静态分析时，可以使用 [骑河集成](../qihe/)。

## 项目配置影响

### 没有完整配置时

打开一个 Verilog/SystemVerilog 文件后，Vide 可以直接报告当前文件里的语法和解析问题，而不包含跨文件语义诊断。

### 有完整配置时

写好 `vide.toml` 后，Vide 会按 `sources`、`include_dirs`、`defines` 和 `libraries` 组织诊断输入。跨文件语义诊断会更接近真实工程。

## 刷新策略

默认 `vide.diagnostics.update` 是 `onSave`，保存文件时刷新诊断。大型工程建议保持这个默认值，避免输入时反复触发较重分析。需要输入时刷新时，可以在 VS Code Settings 中写：

```json
{
  "vide.diagnostics.update": "onType"
}
```

## 调整诊断规则

把光标放在对应诊断位置，打开灯泡菜单。对于带有可识别诊断代码的 slang 诊断，Vide 会提供写入用户设置或工作区设置的快速修复，例如忽略此类诊断，或把错误降级为警告。

也可以在 VS Code Settings 里手动编辑 `vide.diagnostics.slang.rules`。设置参考见 [Diagnostics 设置](../../vscode-settings/#diagnostics)。
