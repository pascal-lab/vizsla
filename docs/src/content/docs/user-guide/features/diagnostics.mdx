## 什么是诊断

诊断就是 Vide 让编辑器显示的报错、警告和提示。它会出现在 VS Code 的 `问题` 面板、编辑器下划线和对应代码位置附近，帮助你发现语法错误、解析错误，以及项目分析发现的语义问题。

## 项目配置影响

打开一个 Verilog/SystemVerilog 文件后，Vide 可以直接报告当前文件里的语法和解析问题。跨文件语义诊断需要工作区里有写好的 `vide.toml`，这样 Vide 才知道哪些源文件、头文件、宏分支和依赖属于同一个工程。配置方式见 [配置第一个项目](../../first-project/) 和 [项目配置参考](../../project-configuration/)。

## 刷新策略

默认 `vide.diagnostics.update` 是 `onSave`，保存文件时刷新诊断。大型工程建议保持这个默认值，避免输入时反复触发较重分析。需要输入时刷新时，可以在 VS Code Settings 中写：

```json
{
  "vide.diagnostics.update": "onType"
}
```

## 调整诊断规则

把光标放在对应诊断位置，打开灯泡菜单。对于带有可识别诊断代码的 slang 诊断，Vide 会提供写入用户设置或工作区设置的快速修复，例如忽略此类诊断，或把错误降级为警告。

也可以在 VS Code Settings 里手动编辑 `vide.diagnostics.slang.rules`。完整写法见 [Diagnostics 设置参考](../../vscode-settings/#diagnostics)。
