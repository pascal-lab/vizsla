## 什么是定义跳转

定义跳转让你从一个符号的使用位置跳到它对应的声明或定义。最常见的场景是从模块实例跳到模块定义，从端口连接跳到目标端口，或者从信号使用位置跳到对应声明。

你可以通过 VS Code 的 `转到定义`、`查看定义`、`转到声明` 和 `转到类型定义` 使用该功能。其中，`转到声明` 会优先返回声明位置；没有单独声明位置时，会使用定义跳转结果。

## 项目配置影响

### 无完整配置时

Vide 会根据当前文件和工作区里扫描到的 Verilog/SystemVerilog 文件提供基础跳转。小工程或临时阅读代码时，这通常已经够用；但 include、宏分支、库边界和同名模块可能无法完整判断。

例如，一个实例名能对应多个同名模块时，Vide 只有在扫描结果里能找到就近且唯一的候选时才会跳转。无法判断唯一目标时，编辑器里会出现提示，提醒你补充项目配置。相关规则见 [分析模型](../../../advanced-guide/parsing-and-analysis/#跳转和同名模块)。

### 有完整配置时

写好 `vide.toml` 后，Vide 会按 `sources`、`libraries`、`include_dirs`、`defines` 和 `top_modules` 描述的工程结构解析目标。跨文件跳转、宏分支下的目标和同名模块边界都会更准确。
