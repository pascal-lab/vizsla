## 什么是重命名

重命名会把同一个 SystemVerilog 符号的声明和引用一起更名。它比手动搜索替换更安全，因为 Vide 会先判断当前位置对应的是哪个符号，再生成工作区编辑，避免把同名但无关的符号一起改掉。

## 项目配置影响

### 没有完整配置时

没有 `vide.toml` 时，Vide 仍然可以重命名那些可编辑范围明确落在当前文件内的符号；如果符号的自然作用域可能扩展到其他文件，Vide 会在应用编辑前停下。这样可以避免在工程范围仍然隐含时误改文件。

### 有完整配置时

有完整配置时可以体验准确的重命名功能。

## 使用边界

如果当前位置不能解析为唯一符号，或者符号的可编辑范围超出了 Vide 当前能安全证明的边界，VS Code 不会进入重命名流程。遇到这种情况，通常先检查 `vide.toml` 是否包含相关源文件、include 目录、宏定义和依赖库。

局部符号默认只会在它实际可见的作用域里重命名。只有在需要调整局部符号跨作用域搜索策略时，才需要查看 `vide.scope.visibility`。完整参考见 [Scope 设置](../../vscode-settings/#scope)。
