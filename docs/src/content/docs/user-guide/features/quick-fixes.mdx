## 什么是自动重构

自动重构用于处理端口连接、参数赋值和数字字面量进制转换这类重复但容易出错的编辑工作。在 VS Code 中，你可以通过 <Codicon name="lightbulb" label="灯泡菜单图标" color="#facc15" /> 灯泡菜单触发这些操作，这个入口也叫 Code Actions。

## 支持的操作

把光标放到对应实例、数字 literal 或诊断附近，打开灯泡菜单。Vide 目前支持这些常见操作：

- `Fill connections`：补齐缺失端口连接。
- `Fill parameters`：补齐缺失参数赋值。
- `Convert ordered port connections to named connections`：把有序端口连接转换为命名连接。
- `Convert ordered parameter assignments to named assignments`：把有序参数赋值转换为命名赋值。
- `Remove empty port connections`：清理多余空端口连接。
- `Add explicit empty port connection`：把 `.port` 改成显式 `.port()`。
- `Add empty instance port list`：给没有端口列表的实例补上空端口列表。
- `Convert literal to binary/octal/decimal/hexadecimal`：转换整数 literal 进制。

## 项目配置影响

### 没有完整配置时

当前文件里的实例和数字字面量可以使用部分自动重构。目标模块在别的文件里时，端口和参数相关操作只有在 Vide 扫描并解析到目标模块后才会出现。

### 有完整配置时

写好 `vide.toml` 后，Vide 会按 `sources`、`include_dirs`、`defines` 和 `libraries` 查找目标模块。补齐端口、补齐参数和转换连接写法会按真实工程结构生成结果。

诊断驱动的快速修复还需要对应诊断已经出现在当前文件里。
