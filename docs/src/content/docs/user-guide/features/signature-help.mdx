## 什么是签名提示

签名提示会在实例参数赋值列表和端口连接列表中显示目标模块的参数或端口信息。它和补全不同：补全是在当前位置插入候选项，签名提示用于确认当前正在填写哪个参数或端口。

## 什么时候出现

- 在参数赋值列表 `#(...)` 中，显示目标模块参数。
- 在端口连接列表 `(...)` 中，显示目标模块端口。
- 光标移动到不同参数或端口位置时，当前项会随之更新。

## 项目配置影响

### 没有完整配置时

如果目标模块就在当前文件里，签名提示可以直接显示。目标模块在别的文件里时，只有 Vide 扫描并解析到该模块后，才能显示对应端口和参数。

### 有完整配置时

写好 `vide.toml` 后，签名提示会按 `sources`、`include_dirs`、`defines` 和 `libraries` 解析目标模块。跨文件实例的参数和端口位置会更准确。

## 调整显示

如果只想显示参数相关签名提示，可以开启：

```json
{
  "vide.signature.help.params.only": true
}
```

完整参考见 [Signature Help 设置](../../vscode-settings/#signature-help)。
