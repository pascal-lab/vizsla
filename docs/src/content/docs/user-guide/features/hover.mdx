## 什么是悬停信息

悬停信息是在光标附近显示的轻量解释。它能够在不离开当前文件的情况下确认目标模块接口、端口方向、参数声明、任务/函数签名，或查看数字字面量的解释结果。

Vide 会根据当前符号的解析结果生成 VS Code hover 内容，内容基于 Markdown 展示，层次清晰易读。

## 项目配置影响

悬停信息需要解析当前位置对应的符号。当前文件里的声明可以直接显示；跨文件模块、端口和参数信息需要目标文件已经被 Vide 解析到。写好 `vide.toml` 后，悬停会按真实工程结构解析 include、宏分支和库文件。

## 相关功能

如果悬停里显示的目标正是你要查看的代码，可以继续使用 [定义跳转](../navigation/) 进入目标文件；如果要确认它还有哪些使用位置，使用 [引用搜索](../references/)。
