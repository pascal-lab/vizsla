## 什么是引用搜索

引用搜索能够找到选中的符号在哪里被使用。在 RTL 工程里，它可以用于追模块实例、端口连接、参数和信号的使用位置，也可以配合定义跳转确认一个连接关系是否完整。

Vide 会响应 VS Code 的 `查找所有引用` 和 `查看引用`。结果会去重后显示在 VS Code 的引用视图里。

## 项目配置影响

引用搜索的范围来自 Vide 当前解析到的文件集合。没有 `vide.toml` 时，Vide 会在工作区扫描结果中查找引用；写好 `vide.toml` 后，引用搜索会按 `vide.toml` 描述的工程范围执行。

## 相关设置

模块声明上方的实例数量提示由 `vide.lens.instantiations.enable` 控制，它使用同一套引用解析结果统计模块被实例化的位置。局部符号是否能跨作用域搜索由 `vide.scope.visibility` 控制；常规项目保持默认值即可。

完整参考见 [Lens 设置](../../vscode-settings/#lens) 和 [Scope 设置](../../vscode-settings/#scope)。

## 和重命名的关系

引用搜索和 [重命名](../rename/) 使用同一套符号解析结果。区别是引用搜索只展示位置，重命名会生成工作区编辑；因此重命名对工程范围要求更严格。
