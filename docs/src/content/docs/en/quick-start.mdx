Follow these steps to start using Vizsla quickly.

## 1. Install the Extension

Search for the display name `Vizsla` in the VS Code Extensions view and install it.

![vizsla in extension market](../assets/quick-start/extension-marketplace.png)

## 2. Open a Project Directory

Open the directory that contains your RTL source code in VS Code.

Vizsla uses `vizsla.toml` as the project configuration file. If the workspace does not have a default project configuration file, the extension shows a prompt asking whether to create one.

![notification on creating manifest](../assets/quick-start/create-manifest-notification.png)

The default project configuration file looks like this:

```toml
#:schema https://pascal-lab.github.io/vizsla/schemas/v1/vizsla.schema.json
sources = []

# include_dirs = ["include"]
# defines = ["SYNTHESIS"]
# top_modules = ["top"]
# libraries = ["../common_cells"]
# exclude = ["build/**"]
```

> The default configuration is a conservative starting point: Vizsla will not scan the whole project or run semantic diagnostics that need full project information. To let Vizsla understand your project structure, add your source directories to `sources`, for example `sources = ["rtl/**"]`. If your project uses include files or macros, add `include_dirs`, `defines`, `libraries`, or `top_modules` as needed.

## 3. Check the Status Bar

After the extension activates, the right side of the VS Code status bar shows a status item named `Vizsla`. The item usually displays `Vizsla`; it shows a spinner while starting or stopping, a warning icon when project configuration is missing, and an error icon when the server or project configuration fails. Hover over the status item to see the current details.

![status bar](../assets/quick-start/status-bar.png)

Click the status item to open the status menu. From there, you can open or create the project configuration file, reload project configuration, restart the language server, profile diagnostic performance, or open the `Vizsla Language Server` output channel.

![status menu](../assets/quick-start/status-menu.png)

## 4. Open a Verilog/SystemVerilog File

Open a `.v`, `.vh`, `.sv`, `.svh`, or `.svi` file. VS Code should recognize it as Verilog or SystemVerilog and enable syntax highlighting and language services.

![example](../assets/quick-start/editor-example.png)

## 5. Try the Core Features

You can verify features in this order:

1. Write an obvious syntax error and check diagnostics in the `Problems` panel.
2. Run `Go to Definition` or `Go to Declaration` on a module name, signal name, or instance name.
3. Trigger completion inside instance port connections, parameter assignments, expressions, or preprocessor positions.
4. Hover over a symbol to view its information.
