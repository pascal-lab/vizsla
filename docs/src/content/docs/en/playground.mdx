The playground loads the same editor, example projects, and WebAssembly language service as the standalone playground. To keep regular documentation pages fast, these examples are loaded on demand: the first example starts when it approaches the viewport, and the rest load only after you click them.

The first load downloads Monaco, the Vizsla WASM engine, and VS Code grammar assets. Later examples on the same page reuse the browser cache. Regular documentation pages only link here and do not download these assets ahead of time.

## Embed State

Documentation pages can set the initial state for each example so readers land directly on the feature being demonstrated:

| Prop | Effect |
| --- | --- |
| `scenario` | Selects the example project. |
| `activeFile` | Opens a specific workspace file. |
| `cursorLine` / `cursorColumn` | Places the cursor at a 1-based line and column. |
| `selection` | Selects a range in `line:column-line:column` format. |
| `diagnosticsOpen` | Opens the diagnostics panel by default. |
| `load` | `click` loads after a click, while `visible` loads near the viewport. |
