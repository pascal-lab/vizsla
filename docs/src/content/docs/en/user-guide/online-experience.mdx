Vide provides several small RTL projects that run directly in the browser. Without extra configuration, you can try symbol navigation, code insight, precise completion, and automatic refactoring.

When an example first loads, wait for the circular indicator in the upper-right corner of the editor to turn green. Then start from the selected code position. You can edit RTL, switch files, open the diagnostics panel, and use the reset button in the toolbar to restore the initial state.

## Symbol Navigation

In traditional Verilog project development, finding where a variable is defined or used often means running a global Ctrl+F string search. That also pulls in similar names and same-named symbols from unrelated scopes, which quickly becomes noisy.

In Vide, go to definition, find references, and document symbols help you move between modules, ports, and registers without leaving the current context, so RTL connections are easier to trace.

This example demonstrates go to definition and find references.

### Steps

1. Load the example. `u_and_gate` is already selected in `top.v`.
2. Run `Go to Definition` on `u_and_gate` or `and_gate` to jump to `and_gate.v`.
3. Return to `top.v` and inspect references or highlights on `gate_out`.

## Code Insight

In a typical editing flow, checking module signatures and port connections often means jumping back and forth between files. Even converting a numeric literal between bases can require an external tool. These interruptions make it harder to stay focused on the RTL design itself.

Vide provides hover information and inlay hints at the edit location, so you can inspect modules, literals, and port connections in one window with fewer context switches.

This example demonstrates hover information for module signatures and instances, plus inlay hints for port names.

### Steps

1. Load the example and read the port hints next to each `u_mux` connection.
2. Hover over `mux2` or `u_mux` to inspect module and instance information.
3. Open the `mux2` definition and compare its port list with the instance connections.

## Precise Completion

Writing RTL without completion depends heavily on remembering language syntax and project structure. If completion only performs string matching, the candidate list is easily polluted by similar symbols and does little to improve editing speed.

Vide's completion understands the current code context. It provides more project-aware suggestions for instantiations, port connections, and other editing positions, and uses snippets for structured completion.

### Steps

1. Load the example and place the cursor inside the `adder8 u0(...)` port list.
2. Delete a port name, or type `.` on a new line and trigger completion.
3. Select a port candidate, then fill in the signal inside the parentheses.

## Automatic Refactoring

Refactoring RTL often involves repetitive work, such as filling missing ports one by one during module instantiation, converting ordered port connections to named connections, or renaming symbols across declarations and uses. These are exactly the mechanical details an editor can handle accurately.

Vide provides automatic refactoring and rename support, so those mechanical details can be handled by the editor.

This example demonstrates automatic completion for missing port connections and rename.

### Steps

1. Load the example, then manually remove one port connection from `u_counter`.
2. Place the cursor near the `u_counter` instance and open the code action menu to complete the missing connection.
3. Rename `smp_cnt` and check that the declaration, connection, and usage update together.
