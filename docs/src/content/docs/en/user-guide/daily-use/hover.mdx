## What It Is

Hover is a lightweight explanation shown near the cursor. It is useful when you want to check a module interface, port direction, parameter declaration, task/function signature, or number literal without leaving the current file.

Vide generates VS Code hover content from the resolved symbol at the current position. When the editor supports Markdown hover, Vide formats the content for readability.

## Project Configuration

Hover needs Vide to resolve the symbol at the current position. Declarations in the current file can be shown directly; cross-file modules, ports, and parameters require the target file to be resolved. After `vide.toml` is configured, hover uses the real include paths, macro branches, and libraries.
