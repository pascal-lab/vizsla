## What It Is

Find References answers “where is this symbol used?” In RTL projects, it is useful for tracing module instances, port connections, parameters, and signals, and for checking whether a connection relationship is complete.

Vide responds to VS Code's `Find All References` and `Peek References`. Results are deduplicated before they are shown in the references view.

## Project Configuration

The reference search scope comes from the files Vide has resolved. Without `vide.toml`, Vide searches workspace scan results. After `vide.toml` is configured, reference search follows the project scope described by `sources`, `libraries`, `include_dirs`, and `defines`.

## Relationship to Rename

[Rename](../rename/) uses the same symbol resolution result. Find References only shows locations; rename generates workspace edits, so it has stricter scope requirements.
