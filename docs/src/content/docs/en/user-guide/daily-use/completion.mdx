## Completion Has Too Few Results

In single-file scenarios, completion usually only uses the current file and whatever has already been indexed. Port, parameter, and instance-related completion needs the target module to resolve; includes, macros, and library dependencies also need to be represented in project configuration.

If the instance module name itself does not jump to definition, first troubleshoot project configuration in [Navigation and Reading](../navigation/#definition-does-not-jump). For call-style hints inside instance parameter and port lists, see [Signature Help](../signature-help/).
