Formatting entry points include VS Code's `Format Document`, `Format Selection`, and on-type formatting when pressing Enter. The only current formatter backend is `verible`, which calls the external `verible-verilog-format` executable; Vide does not bundle that tool.

`vide.formatter.*` and `vide.formatting.*` are VS Code Settings, not `vide.toml` fields. Search for `Vide Formatter` in the Settings UI, or write these keys in user or workspace `settings.json`. See [Formatter and Formatting](../../../advanced-guide/vscode-settings/#formatter-and-formatting) for the full reference.

## Configure verible

`vide.formatter.provider` currently supports `verible`. When `vide.formatter.path` is empty, Vide looks for `verible-verilog-format` on the `PATH` seen by the VS Code process; when the path is set, Vide invokes that executable directly:

```json
{
  "vide.formatter.path": "D:\\tools\\verible\\verible-verilog-format.exe"
}
```

## Formatting Commands

`Format Document` sends the whole file to `verible-verilog-format`. `Format Selection` sends the selected line range to the same formatter. Both commands use the current `vide.formatter.path` and `vide.formatter.args`; `vide.formatting.indent.width` is passed as `--indentation_spaces`.

## Enter Behavior

Enter assistance is controlled by `vide.formatting.on.enter`, and comment handling is controlled by `vide.formatting.in.comments`. Projects that only want manual formatting can disable Enter behavior while keeping `Format Document` and `Format Selection`.
