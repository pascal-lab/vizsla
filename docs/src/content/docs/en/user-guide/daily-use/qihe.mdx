Run `Vide: Run Qihe Analysis` to analyze the current local Verilog/SystemVerilog file with Qihe. The command is available only for `file:` URIs whose extension is `.v`, `.vh`, `.sv`, `.svh`, or `.svi`.

Qihe run options are VS Code Settings. Search for `Vide Qihe` in the Settings UI, or write `vide.qihe.*` keys in user or workspace `settings.json`; see [Qihe](../../../advanced-guide/vscode-settings/#qihe) for the full reference.

## How to Run It

1. Open the RTL file you want to analyze in VS Code.
2. Click the <Codicon name="beaker" label="Run Qihe Analysis icon" /> beaker icon in the editor title bar, or run `Vide: Run Qihe Analysis` from the Command Palette.
3. A separate <Codicon name="beaker" label="Qihe status icon" /> `Qihe` status item appears while the run is active.
4. After a run, click that status item or the `Show Qihe Output` action in the notification to open the `Vide Qihe` output channel.

## Where Arguments Come From

By default, Vide derives the Qihe compile mode, top module, include directories, and macro definitions from the current `vide.toml` project manifest.

<ThinLinkCard

When your project already manages Qihe arguments through scripts, disable automatic derivation and configure explicit arguments:

```json
{
  "vide.qihe.autoConfigureArgsFromManifest": false,
  "vide.qihe.compileArgs": ["--mode", "sv", "--", "-I", "include"],
  "vide.qihe.runArgs": ["-g", "std"]
}
```

Use `vide.qihe.command` when you need to point VS Code at a specific Qihe executable. On Windows, VS Code launched from the desktop may see a different `PATH` than your terminal; an absolute path in workspace settings pins the Qihe version used by the extension.

## Run Scope

This command targets local saved RTL files: the target must be a `file:` URI, and its extension must be `.v`, `.vh`, `.sv`, `.svh`, or `.svi`. Remote files, virtual documents, and unsaved buffers are not Qihe analysis targets.

## Output Contents

The `Vide Qihe` output channel records the target file, compile arguments, run arguments, Qihe stdout/stderr, and final result details. Include directories, macro definitions, and top modules come from [Project Configuration](../../project-configuration/) by default; the Qihe executable is selected by `vide.qihe.command`.

For why Qihe sometimes uses project analysis and sometimes falls back to a single-file input, see [Qihe Project Analysis](../../../advanced-guide/parsing-and-analysis/#qihe-project-analysis).
