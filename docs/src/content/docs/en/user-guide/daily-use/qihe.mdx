Run `Vide: Run Qihe Analysis` to analyze the current local Verilog/SystemVerilog file with Qihe. The command is available only for `file:` URIs whose extension is `.v`, `.vh`, `.sv`, `.svh`, or `.svi`.

Related settings reference: [Qihe](../../../advanced-guide/vscode-settings/#qihe).

## How to Run It

1. Open the RTL file you want to analyze in VS Code.
2. Click the <Codicon name="beaker" label="Run Qihe Analysis icon" /> beaker icon in the editor title bar, or run `Vide: Run Qihe Analysis` from the Command Palette.
3. A separate <Codicon name="beaker" label="Qihe status icon" /> `Qihe` status item appears while the run is active.
4. After a failure, click that status item or the `Show Qihe Output` action in the error notification to open the `Vide Qihe` output channel.

## Where Arguments Come From

By default, Vide derives the Qihe compile mode, top module, include directories, and macro definitions from the current `vide.toml` project manifest.

If your project already manages Qihe arguments through scripts, disable automatic derivation and configure explicit arguments:

```json
{
  "vide.qihe.autoConfigureArgsFromManifest": false,
  "vide.qihe.compileArgs": ["--mode", "sv", "--", "-I", "include"],
  "vide.qihe.runArgs": ["-g", "std"]
}
```

Use `vide.qihe.command` when you need to point VS Code at a specific Qihe executable. On Windows, VS Code launched from the desktop may see a different `PATH` than your terminal; use an absolute path when unsure.

## The Command Is Unavailable or Does Nothing

Confirm that the target is a local file, not a remote file, virtual document, or unsaved buffer. Then confirm the extension is a Verilog/SystemVerilog extension. If the command is still unavailable, focus the actual RTL editor tab and try again.

## Analysis Fails

Open the `Vide Qihe` output channel and read the target file, compile arguments, run arguments, Qihe stdout/stderr, and final failure details in order. If the failure mentions includes, macros, or top modules, fix [Project Configuration](../../project-configuration/) first. If the failure is about launching Qihe, set `vide.qihe.command` to an absolute path.

For why Qihe sometimes uses project analysis and sometimes falls back to a single-file input, see [Qihe Project Analysis](../../../advanced-guide/parsing-and-analysis/#qihe-project-analysis).
