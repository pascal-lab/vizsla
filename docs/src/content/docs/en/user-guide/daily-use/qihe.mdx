Run `Vide: Run Qihe Analysis` to analyze the current local Verilog/SystemVerilog file with Qihe. The command is available only for `file:` URIs whose extension is `.v`, `.vh`, `.sv`, `.svh`, or `.svi`.

Related settings reference: [Qihe](../../../advanced-guide/vscode-settings/#qihe).

## How to Run It

1. Open the RTL file you want to analyze in VS Code.
2. Run `Vide: Run Qihe Analysis` from the command palette, editor title, or file menu.
3. A separate `Qihe` status item appears while the run is active.
4. After a failure, click that status item or the `Show Qihe Output` action in the error notification to open the `Vide Qihe` output channel.

## Where Arguments Come From

By default, Vide derives the Qihe compile mode, top module, include directories, and macro definitions from the current project manifest. The recommended file name is `vizsla.toml`; the legacy `vizsla_config.toml` name is still supported but deprecated.

If your project already manages Qihe arguments through scripts, disable automatic derivation and configure explicit arguments:

```json
{
  "vizsla.qihe.autoConfigureArgsFromManifest": false,
  "vizsla.qihe.compileArgs": ["--mode", "sv", "--", "-I", "include"],
  "vizsla.qihe.runArgs": ["-g", "std"]
}
```

Use `vizsla.qihe.command` when you need to point VS Code at a specific Qihe executable. On Windows, VS Code launched from the desktop may see a different `PATH` than your terminal; use an absolute path when unsure.

## The Command Is Unavailable or Does Nothing

Confirm that the target is a local file, not a remote file, virtual document, or unsaved buffer. Then confirm the extension is a Verilog/SystemVerilog extension. If the command is still unavailable, focus the actual RTL editor tab and try again.

## Analysis Fails

Open the `Vide Qihe` output channel and read the target file, compile arguments, run arguments, Qihe stdout/stderr, and final failure details in order. If the failure mentions includes, macros, or top modules, fix [Project Configuration](../../project-configuration/) first. If the failure is about launching Qihe, set `vizsla.qihe.command` to an absolute path.

For why Qihe sometimes uses project analysis and sometimes falls back to a single-file input, see [Qihe Project Analysis](../../../advanced-guide/parsing-and-analysis/#qihe-project-analysis).
