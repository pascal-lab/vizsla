Vide provides code actions near instance ports, parameters, selected diagnostics, and symbols that can be renamed. Place the cursor on the relevant code and open the VS Code lightbulb menu to see the available fix or refactor.

Quick fixes mostly fall into two groups: repairs that complete or rewrite ports/parameters from diagnostics or instance structure, and local refactors that do not require diagnostics, such as integer literal base conversion. Renameable symbols use VS Code's `Rename Symbol` command.

Diagnostic quick-fix rules are VS Code Settings. Search for `Vide Diagnostics` in the Settings UI, or write `vide.diagnostics.*` keys in user or workspace `settings.json`; see [Diagnostics](../../../advanced-guide/vscode-settings/#diagnostics) for the full reference.

## Supported Actions

Place the cursor near the instance, numeric literal, or diagnostic and open the lightbulb menu. Vide currently supports these common fixes and rewrites:

- Missing port connections: use `Fill connections`.
- Parameters without values: use `Fill parameters`.
- Mixed ordered and named port connections: use `Convert ordered port connections to named connections`, or `Remove empty port connections` for extra empty connections.
- Mixed ordered and named parameter assignments: use `Convert ordered parameter assignments to named assignments`.
- A `.port` shorthand missing explicit `()`: use `Add explicit empty port connection`.
- An instance with no port list: use `Add empty instance port list`.
- Integer literal base is inconvenient: use `Convert literal to binary`, `Convert literal to octal`, `Convert literal to decimal`, or `Convert literal to hexadecimal` near the number.

:::note[Project scope]
Instance port and parameter actions need the target module to resolve. In cross-file projects, `sources`, `include_dirs`, `defines`, and `libraries` decide which modules Vide can see.
:::

## Rename Scope

Rename depends on symbol resolution. `Find References` and `Rename Symbol` use the same project view; after changing project configuration, run `Vide: Reload Project Configuration` to refresh references and rename scope.
