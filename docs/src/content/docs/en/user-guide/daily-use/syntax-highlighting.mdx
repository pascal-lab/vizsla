## What It Is

Syntax highlighting marks keywords, comments, numbers, strings, and basic syntax structures. It does not require full project analysis; it works as soon as you open a Verilog/SystemVerilog file.

The VS Code extension registers these file extensions:

| Language | File extensions |
| --- | --- |
| Verilog | `.v`, `.vh` |
| SystemVerilog | `.sv`, `.svh`, `.svi` |

## Relationship to the Language Server

Basic syntax highlighting, comment rules, bracket matching, and file icons are provided directly by the VS Code extension. After the language server starts, definition, references, hover, completion, diagnostics, semantic highlighting, inlay hints, outline, folding, signature help, and selection ranges become available.

## Project Configuration

Syntax highlighting itself does not depend on `vide.toml`. Project configuration affects semantic features such as cross-file navigation, macro-branch selection, semantic highlighting, and semantic diagnostics. Header files `.vh`, `.svh`, and `.svi` still get basic editing when opened directly; add the including source files to `sources` and the header directories to `include_dirs` to analyze them in the real project context.

## Related Settings

Use [Files settings](../../vscode-settings/#files) to exclude large generated directories from Vide scanning. This affects the files Vide scans, but it does not change VS Code's basic highlighting for an already opened file.
