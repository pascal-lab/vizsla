## What Diagnostics Are

Diagnostics are errors, warnings, and hints that Vide asks the editor to show. They appear in the VS Code Problems panel, as editor underlines, and near the affected code, helping you find syntax errors, parse errors, and semantic issues found by project analysis.

## Project Configuration

After you open a Verilog/SystemVerilog file, Vide can report syntax and parse issues in that file. Cross-file semantic diagnostics need a configured `vide.toml` in the workspace so Vide knows which source files, headers, macro branches, and dependencies belong to the same project. See [Configure the First Project](../../first-project/) and [Project Configuration Reference](../../project-configuration/).

## Refresh Strategy

The default `vide.diagnostics.update` value is `onSave`, so diagnostics refresh when files are saved. Large projects should usually keep this default to avoid running heavier analysis on every edit. To refresh while typing, set:

```json
{
  "vide.diagnostics.update": "onType"
}
```

## Diagnostic Rules

Place the cursor on a diagnostic and open the lightbulb menu. For slang diagnostics with recognizable codes, Vide can write quick fixes into user or workspace settings, such as ignoring that diagnostic or lowering an error to a warning.

You can also edit `vide.diagnostics.slang.rules` manually in VS Code Settings. See [Diagnostics settings](../../vscode-settings/#diagnostics) for the full reference.
