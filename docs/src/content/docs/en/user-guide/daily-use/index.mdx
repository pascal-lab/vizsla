Start from everyday tasks: recognizing RTL files, reading diagnostics, navigating code, completing and refactoring, formatting, and running Qihe from VS Code.

Configuration lives in two places:

- Project structure belongs in the workspace `vide.toml`: source files, include directories, macro definitions, and library dependencies.
- Editor behavior belongs in VS Code Settings: diagnostic refresh, inlay hints, semantic highlighting, formatting, Qihe commands, and other `vide.*` settings. Press `Ctrl+,` and search for `Vide`, or run `Preferences: Open User Settings (JSON)` / `Preferences: Open Workspace Settings (JSON)` and add the setting keys there.

See [VS Code Settings Reference](../../advanced-guide/vscode-settings/) for the full `vide.*` reference.
