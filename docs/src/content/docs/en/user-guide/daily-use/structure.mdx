Structure help reduces the need to jump back and forth while reading RTL:

- Port connection inlay hints: show target port names beside instance connections.
- Parameter assignment inlay hints: show target parameter names in parameter lists.
- End-structure inlay hints: show the matching structure name near `endmodule` and similar endings.
- Semantic highlighting: distinguishes port direction, clock/reset ports, and read/write positions when the theme supports it.
- Instance count lens: shows instantiation counts above module declarations.
- Outline, folding, and expand/shrink selection: help you browse file structure.

Related settings reference: [Inlay Hints](../../../advanced-guide/vscode-settings/#inlay-hints), [Lens](../../../advanced-guide/vscode-settings/#lens), [Semantic Tokens](../../../advanced-guide/vscode-settings/#semantic-tokens).

## Outline, Folding, and Semantic Selection

The outline and breadcrumbs come from VS Code's symbol view, so they are useful for jumping between modules, interfaces, functions/tasks, generate blocks, and declarations. Folding ranges are provided by the language server from syntax structure, covering module bodies, block statements, declaration lists, and similar regions.

![VS Code outline showing module, parameter, port, register, and function symbols](../../../assets/features/document-symbol.png)

_The outline shows modules, parameters, ports, declarations, and functions from an RTL file in a hierarchy._

Use VS Code's `Expand Selection` / `Shrink Selection` commands for semantic selection. Vide expands by syntax nodes, which is more useful than plain text expansion when selecting a port list, expression, instance, or whole construct.

## Adjust Display

Search for `Vide inlay hints` in VS Code Settings and toggle port connection, parameter assignment, and end-structure hints separately. Semantic highlighting also depends on your current theme styling these token types.

:::note[Project scope]
Port/parameter hints and instance count lenses depend on target-module resolution. With only one file open, or with `sources = []`, these results are computed from the project view currently visible to Vide; add the related source files to `sources`, then run `Vide: Reload Project Configuration`.
:::
