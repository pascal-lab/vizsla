Structure help reduces the need to jump back and forth while reading RTL:

- Port connection inlay hints: show target port names beside instance connections.
- Parameter assignment inlay hints: show target parameter names in parameter lists.
- End-structure inlay hints: show the matching structure name near `endmodule` and similar endings.
- Semantic highlighting: distinguishes port direction, clock/reset ports, and read/write positions when the theme supports it.
- Instance count lens: shows instantiation counts above module declarations.
- Outline, folding, and expand/shrink selection: help you browse file structure.

Related settings reference: [Inlay Hints](../../../advanced-guide/vscode-settings/#inlay-hints), [Lens](../../../advanced-guide/vscode-settings/#lens), [Semantic Tokens](../../../advanced-guide/vscode-settings/#semantic-tokens).

## Inlay Hints Are Too Noisy or Missing

Search for `Vide inlay hints` in VS Code Settings and toggle port connection, parameter assignment, and end-structure hints separately. Port and parameter hints depend on resolving the target module; if hints are missing, first check that the instance module name can jump to definition.

## Instance Counts Do Not Appear

Instance counts need module relationships from the project. With only a single file open, or with `sources = []`, results may be incomplete. Add `sources` and run `Vide: Reload Project Configuration`.

## Colors Do Not Change

Semantic highlighting also depends on your VS Code theme styling those tokens. Switch themes once to verify. If nothing changes, check whether `vide.semantic.tokens.*` settings were disabled.
