Use this page to look up Vide user configuration. User configuration changes language-server behavior, such as when diagnostics refresh, whether annotations are shown, which formatter is used, and which Qihe arguments are passed.

User configuration does not describe which files belong to the project. Source files, include directories, macro definitions, libraries, and top modules belong in `vide.toml`; if you are setting up a project for the first time, start with [Configure the First Project](../first-project/).

## Where to Put It

The same option has slightly different names in different editors.

VS Code users can search for `Vide` in the Settings UI or edit `settings.json` directly. VS Code setting names use a `vide.` prefix:

```json
{
  "vide.diagnostics.update": "onType",
  "vide.inlayHints.port.connection.enable": true
}
```

Editors such as Neovim and Emacs usually start Vide through LSP. LSP is the protocol between the editor and the language server. These editors usually pass the same configuration as initialization options. Initialization options are the JSON object that the editor sends when it starts the language server. This form does not use the `vide.` prefix:

```json
{
  "diagnostics": {
    "update": "onType"
  },
  "inlayHints": {
    "port": {
      "connection": {
        "enable": true
      }
    }
  }
}
```

If you use Neovim, Emacs, or another LSP editor, the advanced guide shows where to put these options and includes complete examples: [User Configuration for LSP Editors](../../advanced-guide/user-configuration/).

## How to Read the Table

The `User config` column uses dot paths to describe positions inside the JSON object. For example, `diagnostics.update` means:

```json
{
  "diagnostics": {
    "update": "onSave"
  }
}
```

The `VS Code name` column shows the same option name in VS Code `settings.json`. For example, `diagnostics.update` maps to `vide.diagnostics.update`.

Start with defaults and only change an option when you have a reason.

## Schema URL

A JSON schema describes the shape of a configuration file. When your editor or config tool supports JSON schema validation, it can offer completion and catch typos such as `diagnostic` instead of `diagnostics` before the server starts.

The user configuration schema URL is:

```text
https://vide.pascal-lab.net/schemas/v1/user-config.schema.json
```

If you keep user configuration in a standalone JSON file, you can add `$schema` at the top and point it to this URL. `$schema` is only for editor completion and validation; Vide ignores it when reading configuration. When writing options directly in Neovim `init_options` or Emacs `:initializationOptions`, you usually do not need `$schema`.

## VS Code Extension Startup Settings

These are not language-server user configuration. They are VS Code extension settings used to start the background language server. Change them only when you need to replace the local `vide` executable, add startup arguments, set a working directory, or debug LSP communication.
