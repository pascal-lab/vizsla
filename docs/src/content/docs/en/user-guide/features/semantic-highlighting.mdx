## What It Is

Syntax highlighting only knows whether text looks like a keyword, number, or comment. Semantic highlighting uses Vide's analysis result to mark port direction, clock/reset ports, and symbol read/write positions. When the active VS Code theme styles semantic tokens, these details appear as colors or font styles.

## Project Configuration

Basic semantic tokens for the current file can be computed directly. Cross-file resolution, macro branches, and include relationships affect target modules, ports, and declarations. After `vide.toml` is configured, semantic highlighting follows the project structure described by `sources`, `include_dirs`, `defines`, and `libraries`.

## Settings

Semantic highlighting also depends on the active VS Code theme. Vide provides two switches:

- `vide.semantic.tokens.port.clk.rst.enable`: controls clock/reset port tokens.
- `vide.semantic.tokens.port.input.output.enable`: controls input/output direction tokens.

See [Semantic Tokens settings](../../vscode-settings/#semantic-tokens) for the full reference.
