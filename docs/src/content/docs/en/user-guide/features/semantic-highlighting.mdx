## What It Is

Syntax highlighting only knows whether a token looks like a keyword, number, or comment. Semantic highlighting uses Vide's parse result so RTL identifiers that look the same can use different styles. When reading code, you can more quickly distinguish clock/reset ports, input/output ports, module instance names, and `typedef` types. In the default display, input ports are bold (for example <strong>data_i</strong>), output ports are bold and underlined (for example <strong><u>data_o</u></strong>), and clock/reset ports use colors that differ from ordinary ports.

## Project Configuration

Basic semantic markings for the current file can be computed directly. Cross-file resolution, macro branches, and include relationships affect target modules, ports, and declarations. After `vide.toml` is configured, semantic highlighting refreshes from the project structure described by `vide.toml`.

## Settings

Semantic highlighting also depends on the active VS Code theme. Vide provides two switches:

- `vide.semantic.tokens.port.clk.rst.enable`: controls clock/reset port tokens.
- `vide.semantic.tokens.port.input.output.enable`: controls input/output direction tokens.

See [Semantic Tokens settings](../../vscode-settings/#semantic-tokens) for the full reference.
