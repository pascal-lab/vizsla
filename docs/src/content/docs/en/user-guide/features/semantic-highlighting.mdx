## What It Is

Syntax highlighting only knows whether a token looks like a keyword, number, or comment. Semantic highlighting further identifies what a name means in code. When reading code, you can more quickly distinguish clock/reset ports, input/output ports, module instance names, and `typedef` types. In the default display, input ports are bold (for example <strong>data_i</strong>), output ports are bold and underlined (for example <strong><u>data_o</u></strong>), and clock/reset ports use colors that differ from ordinary ports.

## Project Configuration

### Without Complete Configuration

Ports, declarations, and types in the current file can receive basic semantic highlighting. Cross-file targets, include files, and macro branches only participate when Vide can scan and resolve them.

### With Complete Configuration

After `vide.toml` is configured, semantic highlighting refreshes from the project structure. Port directions, clock/reset ports, instance names, and type names are identified with the real include files, macro definitions, and libraries.

## Settings

If semantic highlighting does not look different, first confirm that the active VS Code theme supports semantic tokens. Vide provides two switches:

- `vide.semantic.tokens.port.clk.rst.enable`: controls clock/reset port tokens.
- `vide.semantic.tokens.port.input.output.enable`: controls input/output direction tokens.

See [Semantic Tokens settings](../../vscode-settings/#semantic-tokens).
