## What It Is

Syntax highlighting only knows whether a token looks like a keyword, number, or comment. Semantic highlighting goes further by identifying what a name means in the code. When reading code, you can distinguish clock/reset ports, input/output ports, module instance names, and `typedef` types more quickly. In the default display, input ports are bold (for example <strong>data_i</strong>), output ports are bold and underlined (for example <strong><u>data_o</u></strong>), and clock/reset ports use colors that differ from ordinary ports.

## Project Configuration

### Without Complete Configuration

Ports, declarations, and types in the current file can receive basic semantic highlighting. Cross-file targets, include files, and macro branches only participate when Vide can scan and resolve them.

### With Complete Configuration

After `vide.toml` is configured, semantic highlighting refreshes from the project structure. Port directions, clock/reset ports, instance names, and type names are identified using the actual include files, macro definitions, and libraries.

## Settings

If semantic highlighting does not look different, first confirm that the active VS Code theme supports semantic tokens. You can also try another theme to check whether the markings take effect.

Vide provides two switches:

- `vide.semantic.tokens.port.clk.rst.enable`: shows or hides semantic coloring for clock/reset ports.
- `vide.semantic.tokens.port.input.output.enable`: shows or hides semantic styling for input/output port directions.

See [Semantic Tokens settings](../../vscode-settings/#semantic-tokens).
