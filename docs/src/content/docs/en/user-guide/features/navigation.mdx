## What It Is

Go to Definition takes you from a symbol use to the declaration or definition it points at. The common cases are jumping from a module instance to the module definition, from a port connection to the target port, or from a signal use to its declaration.

You can use this feature through VS Code's `Go to Definition`, `Peek Definition`, `Go to Declaration`, and `Go to Type Definition`. `Go to Declaration` prefers declaration locations; when there is no separate declaration location, it uses the definition result.

## Project Configuration

### Without Complete Configuration

Vide uses the current file and Verilog/SystemVerilog files scanned from the workspace to provide basic jumps. This is often enough for small projects or quick code reading, but include paths, macro branches, library boundaries, and duplicate modules may not be fully understood.

For example, if one instance name can match multiple modules with the same name, Vide only jumps when the scanned files provide a nearby and unique candidate. If no unique target can be determined, the editor shows a hint asking you to add project configuration. See [Analysis Model](../../../advanced-guide/parsing-and-analysis/#navigation-and-duplicate-modules) for details.

### With Complete Configuration

After `vide.toml` is configured, Vide resolves targets from the project structure described by `sources`, `libraries`, `include_dirs`, `defines`, and `top_modules`. Cross-file jumps, targets inside macro branches, and duplicate module boundaries become more accurate.
