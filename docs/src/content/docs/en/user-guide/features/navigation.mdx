## What It Is

Go to Definition takes you from a reference to the declaration or definition it points at. In RTL, the common cases are jumping from a module instance to the module definition, from a port connection to the target port, or from a signal use to its declaration.

Vide wires VS Code's `Go to Definition`, `Peek Definition`, `Go to Declaration`, and `Go to Type Definition` to RTL analysis. `Go to Declaration` prefers declaration ranges; when there is no separate declaration range, it uses the definition result.

## Project Configuration

Definition needs Vide to resolve the target file. Without `vide.toml`, Vide scans the workspace to provide reading-time jumps. After `vide.toml` is configured, targets are resolved through `sources`, `libraries`, `include_dirs`, and `defines`.

Check these fields first:

- `sources` covers the file containing the target module.
- `libraries` includes external dependency projects.
- `include_dirs` contains directories used by `` `include ``.
- `defines` enables the macro branch containing the target code.

## Duplicate Modules

Without project configuration, Vide chooses a nearby and unique candidate for reading. Real projects should handle duplicate module names through explicit project configuration, library boundaries, or build scripts. See [Analysis Model](../../../../advanced-guide/parsing-and-analysis/#navigation-and-duplicate-module-names) for details.
