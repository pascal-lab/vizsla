## What It Is

Find References answers “where is this symbol used?” In RTL projects, it is useful for tracing module instances, port connections, parameters, and signals, and for checking whether connections are complete.

Run VS Code's `Find All References` or `Peek References` on a symbol. Vide searches for its usage locations, deduplicates them, and shows them in the references view.

## Project Configuration

### Without Complete Configuration

Vide searches references in all Verilog/SystemVerilog files scanned from the workspace. This is useful for quick code reading, but results can be incomplete when the project depends on include paths, macro branches, or external libraries.

### With Complete Configuration

After `vide.toml` is configured, reference search follows the project scope described by `sources` and `libraries`, and uses include directories and macro definitions to decide which files participate in analysis.

## Instance Count Lens

The instance-count lens above module declarations is controlled by `vide.lens.instantiations.enable`. It uses reference search capability to count where a module is instantiated.

See [Lens settings](../../vscode-settings/#lens).

## Relationship to Rename

Find References only shows locations and does not modify files. Rename edits declarations and references, so it usually needs `vide.toml` to make the editable project scope explicit.
