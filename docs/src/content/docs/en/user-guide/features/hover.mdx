## What It Is

Hover is a lightweight explanation shown near the cursor. It is useful when you want to check a module interface, port direction, parameter declaration, task/function signature, or numeric literal without leaving the current file.

Vide generates VS Code hover content from the resolved symbol at the current position. The content is shown as Markdown with a readable structure.

## Project Configuration

Hover depends on symbol resolution at the current position. Vide can show declarations in the current file immediately; cross-file modules, ports, and parameters require target-file resolution. After `vide.toml` is configured, hover uses the configured include paths, macro branches, and libraries.
