## What Formatting Is

Formatting keeps indentation, line breaks, and style consistent so the code is easier to read.

Formatting entry points include VS Code's `Format Document`, `Format Selection`, and the automatic indentation behavior when pressing Enter. Formatting currently uses the third-party Verible `verible-verilog-format` tool. Vide does not bundle that tool, so you need to configure the executable yourself.

## Configure verible

Verible is a ChipsAlliance SystemVerilog tool suite that includes a formatter. Vide's formatting support depends on its `verible-verilog-format` executable. Install Verible first, then configure it in VS Code.

<ThinLinkCard
  href="https://github.com/chipsalliance/verible#installation"

In VS Code Settings, when `vide.formatter.path` is empty, Vide looks for `verible-verilog-format` on the `PATH` seen by the VS Code process. When the path is set, Vide invokes that executable directly:

```json
{
  "vide.formatter.path": "D:\\tools\\verible\\verible-verilog-format.exe"
}
```

Use `vide.formatter.args` to pass options to `verible-verilog-format` and adjust formatting behavior.

## Formatting Commands

`Format Document` sends the whole file to `verible-verilog-format`. `Format Selection` sends the selected line range to the same formatter. Both commands use the current `vide.formatter.path` and `vide.formatter.args`; `vide.formatting.indent.width` is passed as `--indentation_spaces`.

## Enter Behavior

Besides manually running `Format Document` and `Format Selection`, Vide can adjust the new line's indentation when you press Enter. This behavior is controlled by `vide.formatting.on.enter`.

If you only want Enter assistance outside comments, disable `vide.formatting.in.comments`. If the project only needs manual formatting, disable `vide.formatting.on.enter` and keep `Format Document` and `Format Selection`.

See [Formatter and Formatting settings](../../vscode-settings/#formatter-and-formatting).
