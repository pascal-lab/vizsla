## What It Is

Rename updates the declaration and references of the same SystemVerilog symbol together. It is safer than manual search-and-replace because Vide first resolves the symbol under the cursor, then generates a workspace edit that avoids unrelated symbols with the same name.

## Project Configuration

### Without Complete Configuration

Without `vide.toml`, Vide only performs rename when all edits are proven to stay inside the current file. If the symbol may affect other files, rename is rejected and Vide asks you to add `vide.toml` to make the editable project scope explicit.

### With Complete Configuration

With complete configuration, rename can work accurately.

## Boundaries

If the position does not resolve to a unique symbol, or if the symbol's editable scope reaches beyond what Vide can trace at all, VS Code does not enter the rename flow. When this happens, first check whether `vide.toml` includes the relevant source files, include directories, macro definitions, and libraries.

Local symbols are renamed only inside the scope where they are actually visible by default. You only need `vide.scope.visibility` when you want to adjust local-symbol cross-scope search behavior.
