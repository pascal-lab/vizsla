## What It Is

Rename changes the declaration and references of the same RTL symbol together. It is safer than manual search-and-replace because Vide first resolves the symbol at the cursor, then generates a VS Code workspace edit, so the changed symbol does not leak beyond its scope and affect other symbols.

## Project Configuration

Rename depends on symbol resolution and an editable project scope. Without `vide.toml`, Vide uses workspace scan results for reading features, but project rename is not enabled. Add the relevant source files to `sources`, then run `Vide: Reload Project Configuration` to refresh the rename scope.

## Boundaries

If the position does not resolve to a unique symbol, or if the target file is outside the editable scope managed by the current project configuration, VS Code does not enter the rename flow. Confirm that [Go to Definition](../navigation/) and [Find References](../references/) resolve the same target first.

Local-symbol scope visibility is controlled by `vide.scope.visibility`; keep the default for normal projects. See [Scope settings](../../vscode-settings/#scope).
