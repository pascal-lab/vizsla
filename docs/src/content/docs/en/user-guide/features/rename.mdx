## What It Is

Rename updates the declaration and references of the same SystemVerilog symbol together. It is safer than manual search-and-replace because Vide first resolves the symbol under the cursor, then generates a workspace edit that avoids unrelated symbols with the same name.

## Project Configuration

### Without Complete Configuration

Without `vide.toml`, Vide can use workspace scan results for reference search, but project-wide rename is not enabled. This avoids modifying files when the project scope is unclear.

### With Complete Configuration

Add the source files that should participate in rename to `sources` or related `libraries`, then run `Vide: Reload Project Configuration`. Vide reloads the project scope, and rename updates declarations and references in those editable files.

## Boundaries

If the position does not resolve to a unique symbol, or if the target file is outside the editable scope managed by the current project configuration, VS Code does not enter the rename flow. When this happens, first check whether `vide.toml` includes the relevant source files, include directories, macro definitions, and libraries.

Local symbols are renamed only inside the scope where they are actually visible by default. You only need `vide.scope.visibility` when you want to adjust local-symbol cross-scope search behavior. See [Scope settings](../../vscode-settings/#scope).
