## What It Is

Completion inserts candidates at the current position. Vide combines syntax position and RTL semantics: target ports in instance port lists, target parameters in parameter lists, visible symbols in expressions, and directives or macro-related candidates in preprocessor positions.

Common trigger positions include instance `#(...)` and `(...)`, expressions, member access `.`, event controls `@(...)`, system tasks after `$`, preprocessor positions after `` ` ``, and integer literal base markers after `'`.

## Project Configuration

In single-file cases, completion uses the current file and indexed content. Port, parameter, and instance completion need the target module to resolve. In cross-file projects, `vide.toml` decides which modules and macro branches Vide can see.

## Related Features

Completion inserts candidates. If you are already inside an instance parameter or port list and need to confirm the current item, use [Signature Help](../signature-help/).
