This is a `vide.toml` project configuration reference. For first-time setup, first use [Configure the First Project](../first-project/) to write a working project configuration file.

## Field Quick Reference

A common small project can use these fields:

```toml
sources = ["rtl/**"]
include_dirs = ["include"]
defines = ["SYNTHESIS"]
top_modules = ["top"]
```

| Field | Type | Purpose |
| --- | --- | --- |
| `sources` | Array of globs, or omitted | Selects project source files. A common value is `["rtl/**"]`. |
| `include_dirs` | Array of paths, or omitted | Include search directories, for example `["include"]`. |
| `defines` | Array of strings | Predefined macros. Supports both `"NAME"` and `"NAME=value"`. |
| `top_modules` | Array of strings | Declares top-level modules for the current project. |
| `libraries` | Array of paths | External libraries or dependency project directories. |
| `exclude` | Array of globs | Filters generated directories, simulation output, or black-box files out of loaded files. |

## Path and Glob Rules for `sources` and `exclude`

`sources` and `exclude` are not plain directory paths. They are path patterns with wildcards, also called globs. The two most common wildcards are:

- `*`: matches names within one directory. For example, `rtl/*.sv` only matches `.sv` files directly under `rtl`.
- `**`: matches across nested directories. For example, `rtl/**` continues matching files inside subdirectories under `rtl`.

:::note
For background, see this [glob overview](https://en.wikipedia.org/wiki/Glob_(programming)).
:::

Vide applies these rules to `sources` and `exclude`:

- Patterns are relative to the workspace root, for example `rtl/**`.
- Use `/` as the path separator. Do not write `rtl\**`.
- Absolute paths and any pattern containing a `..` path component are rejected.
- `*` does not cross directories; use `**` when you need recursive matching.

In `include_dirs` and `libraries`, values are ordinary paths resolved relative to the directory that contains the current project configuration file. `include` and `include/` refer to the same directory; for consistency, this documentation uses `include`.

## Field Details

### sources and include_dirs

`sources` has three common states:

| Form | Result |
| --- | --- |
| Omitted | Vide best-effort indexes the workspace for read features such as navigation, references, and hover, but does not treat it as a fully configured project. |
| `sources = []` | Disables automatic workspace scanning. This is useful when you want to open the project safely and configure it later. |
| `sources = ["rtl/**"]` | Loads matching files with globs and puts them into project analysis. |

Common examples:

```toml
sources = ["rtl/**", "ip/**/*.sv", "tb/**/*.sv"]
```

The current project loader reads these extensions:

- `.v`
- `.sv`
- `.vh`
- `.svh`
- `.svi`
- `.map`

`include_dirs` controls `` `include "file.svh" `` search. If `include_dirs` is not configured explicitly and `sources` is set explicitly, Vide infers a default include directory from `sources`. For example, `sources = ["rtl/**/*.sv"]` uses `rtl` as the default include directory.

If `sources` is omitted, Vide does not infer include directories from best-effort indexing. If `include_dirs = []` is set explicitly, this fallback is not used either.

### defines

Each entry in `defines` is one macro definition:

```toml
defines = [
  "SYNTHESIS",
  "WIDTH=32",
  "MODE=fast sim",
]
```

Macro names must be valid identifiers. Macros with values are parsed as `NAME=value`; `value` may contain spaces.

### libraries

`libraries` declares dependency libraries. Paths are resolved relative to the directory that contains the current project configuration file. Each library path is loaded as another workspace:

- If the library directory contains `vide.toml`, Vide loads the library with that project configuration file.
- If the library directory has no project configuration file, Vide loads it as an unconfigured library.
- Dependencies participate in the current project's analysis, and transitive dependencies are supported.

`libraries` does not replace the current project's own `sources`. The current workspace's RTL should still be declared through `sources`; `libraries` only describes external projects or library directories referenced by the current project.

### exclude

`exclude` uses the same workspace-relative glob syntax described above. It is useful for generated directories, simulation output, cache directories, or black-box stubs:

```toml
exclude = ["build/**", "out/**", "sim/work/**", "**/*_bb.v"]
```

`exclude = ["build"]` only matches the path named `build` itself and does not recursively exclude the directory contents. Use `build/**` for recursive directory exclusion. If you also want VS Code to reduce file watching, configure VS Code's `files.watcherExclude` as well. Vide's `exclude` and VS Code's watcher settings are separate mechanisms.

## Notes

- The project configuration file name must be `vide.toml`.
- Vide does not automatically discover project configuration files from subdirectories. When you open `repo`, it only checks `repo/vide.toml`.
