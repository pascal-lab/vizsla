This page shows how to install Vide and verify that it is working.

## 1. Install the Extension

For day-to-day use, install the stable Marketplace extension.

<ThinLinkCard
  href="https://marketplace.visualstudio.com/items?itemName=pascal-lab.vide"
  title="Visual Studio Marketplace"
  action="Open"
>
  Install stable Vide, extension ID: <code>pascal-lab.vide</code>
</ThinLinkCard>

You can also search for the display name `Vide` or the extension ID `pascal-lab.vide` in the VS Code Extensions view.

![vide in extension market](../../assets/quick-start/extension-marketplace.png)

## 2. Open an RTL Project Directory

Open the folder that contains your RTL source code in VS Code.

If the project contains Verilog/SystemVerilog source files but no `vide.toml` project configuration file, the extension prompts you to create a default configuration. For a quick trial, you can skip this step for now; Vide can still provide some analysis without configuration.

![notification for creating a project configuration file](../../assets/quick-start/create-manifest-notification.png)

We will create the first `vide.toml` in [Configure the First Project](../first-project/).

## 3. Confirm Vide Started

After the extension activates, the right side of the VS Code status bar shows a `Vide` status item. During a normal startup, the item displays `Vide`. Hover over it and check the status details to confirm that Vide has started.

![status bar](../../assets/quick-start/status-bar.png)

:::note
If the `Vide` status item does not appear, reload VS Code and open the project folder. If it still does not appear after restarting, please [contact us](https://github.com/pascal-lab/vide/issues/new).
:::

## 4. Open an RTL File

Open a Verilog `.v`/`.vh` file or a SystemVerilog `.sv`/`.svh`/`.svi` file. VS Code should recognize the language, enable syntax highlighting, and start Vide analysis.

![example](../../assets/quick-start/editor-example.png)

## Next Step

After verifying the installation, use [Configure the First Project](../first-project/) to write your first `vide.toml`, then continue to [Features](../features/).
