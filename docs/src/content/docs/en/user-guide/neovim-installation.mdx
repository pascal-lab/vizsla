This page is for users who already use Neovim's built-in LSP client. It walks you through downloading the Vide language server and configuring Neovim to start it for Verilog/SystemVerilog files, so features such as go-to-definition, completion, hover, and diagnostics show up in the editor UI.

## 1. Download Vide

Choose the language server binary for your platform, and rename it to `vide` (`vide.exe` on Windows):

| Platform | Download |
| --- | --- |
| Linux x64 | [`vide-linux-x64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-linux-x64) |
| Linux arm64 | [`vide-linux-arm64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-linux-arm64) |
| Windows x64 | [`vide-win32-x64.exe`](https://github.com/pascal-lab/vide/releases/latest/download/vide-win32-x64.exe) |
| macOS arm64 | [`vide-darwin-arm64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-darwin-arm64) |
| Alpine/musl x64 | [`vide-alpine-x64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-alpine-x64) |
| Alpine/musl arm64 | [`vide-alpine-arm64`](https://github.com/pascal-lab/vide/releases/latest/download/vide-alpine-arm64) |

On Linux or macOS, run the following (replace `<platform>` with the matching platform name):

```bash
mkdir -p ~/.local/bin
curl -L https://github.com/pascal-lab/vide/releases/latest/download/vide-<platform> -o ~/.local/bin/vide
chmod +x ~/.local/bin/vide
vide --version
```

Windows PowerShell example:

```powershell
New-Item -ItemType Directory -Force "$env:LOCALAPPDATA\vide"
Invoke-WebRequest "https://github.com/pascal-lab/vide/releases/latest/download/vide-win32-x64.exe" -OutFile "$env:LOCALAPPDATA\vide\vide.exe"
& "$env:LOCALAPPDATA\vide\vide.exe" --version
```

After downloading, confirm that `vide --version` runs and returns version information.

You can add the directory containing `vide` (or `vide.exe`) to `PATH` to make the next steps more convenient, but this is optional.

## 2. Find Your Neovim Config File

Next, you need to edit your Neovim configuration. Run these commands inside Neovim to confirm the config directory and the config file currently in use:

```vim
:echo stdpath('config')
:echo $MYVIMRC
```

If the second command prints nothing, you do not have a config file yet. In that case, create `init.lua` under the directory returned by `stdpath('config')`. For example, on Windows the first command usually prints something like:

```text
C:\Users\<user>\AppData\Local\nvim
```

That means your config file should be created at `C:\Users\<user>\AppData\Local\nvim\init.lua`.

## 3. Configure Neovim to Start Vide

Add the following to the `init.lua` file mentioned above:

```lua
vim.filetype.add({
  extension = {
    v = 'verilog',
    vh = 'verilog',
    sv = 'systemverilog',
    svh = 'systemverilog',
    svi = 'systemverilog',
  },
})

vim.lsp.config('vide', {
  cmd = { 'vide' },
  filetypes = { 'verilog', 'systemverilog' },
  root_markers = { 'vide.toml', '.git' },
})

vim.lsp.enable('vide')
```

This configuration does two things:

- It explicitly maps `.v`, `.vh`, `.sv`, `.svh`, and `.svi` to `verilog` or `systemverilog`, so filetype detection does not depend on Neovim version or local defaults.
- It tells Neovim to start the `vide` language server for those filetypes and use `vide.toml` or `.git` as project-root markers.

If `vide` from step 1 is not on `PATH`, use an absolute path in `cmd`, for example:

```lua
cmd = { 'D:/vide/vide.exe' }
```

After opening a file, you can verify the detected filetype with:

```vim
:set filetype?
```

Expected results: `.v`/`.vh` should show `filetype=verilog`, and `.sv`/`.svh`/`.svi` should show `filetype=systemverilog`.

## 4. Prepare Project Configuration

Create `vide.toml` in the project root so Vide knows which source files, include directories, and macros to analyze. See [Configure the First Project](../first-project/) for the first setup.

To tune diagnostics, file watching, inlay hints, or other language-server behavior in Neovim, see [User Configuration for LSP Editors](../../advanced-guide/user-configuration/).

## 5. Verify

Open a `.v` or `.sv` file in your project, then run this in Neovim:

```vim
:lua print(vim.inspect(vim.lsp.get_clients({ name = 'vide' })))
```

If the output contains a `vide` client, the language server has started. You can also directly try go-to-definition, hover, completion, or diagnostics in code.
