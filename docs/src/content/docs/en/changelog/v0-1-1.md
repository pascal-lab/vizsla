## Fixes

- Fixed completion context misclassification. Integer literal base completion, local slang directive completion, and library map prefix completion now use token ranges, reducing interference from VS Code word-based suggestions. ([#18](https://github.com/pascal-lab/vizsla/pull/18))
- Fixed ANSI port inherited port header types, preventing ports with omitted types from losing inherited information in HIR and signature help. ([#24](https://github.com/pascal-lab/vizsla/pull/24))
- Fixed an issue where clearing file content did not synchronize the empty text into LSP analysis state, preventing later diagnostics and semantic results from using stale text. ([#25](https://github.com/pascal-lab/vizsla/pull/25))
- Fixed duplicate file watcher registration so stale `workspace/didChangeWatchedFiles` registrations do not remain after switching workspaces. ([#26](https://github.com/pascal-lab/vizsla/pull/26))
- Fixed a possible slang source manager duplicate buffer allocation and crash when a `.sv` file is both scanned as a workspace source and included via `` `include``. `.sv` files reached through include are treated as include-only units while preserving diagnostic mapping. ([#29](https://github.com/pascal-lab/vizsla/pull/29))
- Fixed borrowed syntax nodes in semantic requests that could dangle after parse cache eviction, reducing crash risk for navigation, hover, references, rename, and related semantic features after cache eviction. ([#31](https://github.com/pascal-lab/vizsla/pull/31))
- Fixed unstable include buffer / VFS file ID mapping across platforms and path spellings, which could cause diagnostics to be lost. ([#32](https://github.com/pascal-lab/vizsla/pull/32))

## Improvements

- Added an integer literal base conversion code action. It can convert between binary, octal, decimal, and hexadecimal while preserving signedness, width, and related literal semantics. ([#21](https://github.com/pascal-lab/vizsla/pull/21))
- Linux x64 VSIX packages are now built on Ubuntu 22.04 runners to reduce dependency on newer glibc/libstdc++; `alpine-x64` and `alpine-arm64` VSIX targets were also added. ([#36](https://github.com/pascal-lab/vizsla/pull/36))

## Internal and Documentation

- Stabilized pull diagnostics tests to reduce uncertainty in asynchronous diagnostics tests. ([#40](https://github.com/pascal-lab/vizsla/pull/40))
- Released VS Code extension `0.1.1`. ([#41](https://github.com/pascal-lab/vizsla/pull/41))
