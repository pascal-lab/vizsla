## Breaking Changes

- **Impact: high.** `sources` / `exclude` are now interpreted as workspace-relative shell globs. Existing `sources = ["rtl"]` configurations should become `sources = ["rtl/**"]` if they intend to include a directory recursively; `exclude = ["build"]` should become `exclude = ["build/**"]`. This only affects projects that configured these fields with directory-style values. ([#69](https://github.com/pascal-lab/vizsla/pull/69))
- **Impact: medium.** When the manifest is missing, empty, or omits `sources`, Vide no longer scans the whole workspace for semantic analysis by default. It enters syntax-only mode instead. Projects that relied on automatic full-workspace scanning will lose semantic diagnostics or cross-file analysis until `sources`, `include_dirs`, and related fields are configured explicitly. ([#47](https://github.com/pascal-lab/vizsla/pull/47))

## Fixes

- Fixed panics triggered by `documentSymbol` / `definition` in some projects. The root cause was expanding `include` incorrectly during pathless/HIR analysis, causing syntax node ranges to not belong to the current file. ([#44](https://github.com/pascal-lab/vizsla/pull/44), [#46](https://github.com/pascal-lab/vizsla/pull/46))
- Fixed diagnostics not refreshing promptly after editing `vizsla_config.toml` / `vizsla.toml`; workspace diagnostics now refresh after manifest reload. ([#63](https://github.com/pascal-lab/vizsla/pull/63))
- Fixed missing completion for system tasks/functions such as `$display`, `$finish`, and `$time` after a standalone `$`. ([#61](https://github.com/pascal-lab/vizsla/pull/61))
- Fixed `vizsla.toml` being incorrectly sent to slang diagnostics as a SystemVerilog file; `vizsla.toml` and `vizsla_config.toml` are now both recognized as project manifests. ([#71](https://github.com/pascal-lab/vizsla/pull/71))
- Fixed LSP trace setting issues, added support for `$/setTrace`, and reduced noisy trace logs. ([#51](https://github.com/pascal-lab/vizsla/pull/51))

## Improvements

- `sources` / `exclude` now support glob patterns such as `*.v`, `src/**/*.sv`, and `*_bb.v`. ([#69](https://github.com/pascal-lab/vizsla/pull/69))
- Added `vizsla.toml` as the recommended manifest file name while keeping `vizsla_config.toml` as a compatible fallback. ([#60](https://github.com/pascal-lab/vizsla/pull/60))
- The VS Code extension can now create a workspace configuration automatically. When it detects an RTL workspace with no configuration file, it can prompt the user to create a default `vizsla.toml`. ([#47](https://github.com/pascal-lab/vizsla/pull/47), [#60](https://github.com/pascal-lab/vizsla/pull/60), [#66](https://github.com/pascal-lab/vizsla/pull/66))
- Added JSON Schema artifacts and generation flow for project configuration files. The default configuration writes a `#:schema` directive. Editing assistance for configuration files is mostly left to TOML tools that support schema directives; Vide itself only reads configuration and refreshes the project model. ([#59](https://github.com/pascal-lab/vizsla/pull/59))
- The `qihe` integration now supports project-level analysis based on workspace / manifest configuration instead of preferring only single-file analysis. ([#45](https://github.com/pascal-lab/vizsla/pull/45))
- Improved hover information: modules, ports, parameters, tasks/functions, and related items now show fuller signatures, and definitions from multiple sources are separated more clearly. ([#72](https://github.com/pascal-lab/vizsla/pull/72))
- Added a parameterized module snippet that completes a module template with `#(...)`. ([#62](https://github.com/pascal-lab/vizsla/pull/62))
- Added semantic token styling support in VS Code. ([#65](https://github.com/pascal-lab/vizsla/pull/65))

## Internal and Documentation

- Refactored the code action module structure into action, diagnostics, collector, handlers, tests, and related files to improve maintainability. ([#22](https://github.com/pascal-lab/vizsla/pull/22))
- Migrated documentation to Astro Starlight and updated configuration documentation. ([#52](https://github.com/pascal-lab/vizsla/pull/52), [#70](https://github.com/pascal-lab/vizsla/pull/70))
- Updated CI to avoid running the same checks twice for PR branches. ([#67](https://github.com/pascal-lab/vizsla/pull/67))
- Released VS Code extension `0.1.2`. ([#74](https://github.com/pascal-lab/vizsla/pull/74))
