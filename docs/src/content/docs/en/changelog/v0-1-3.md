## Indexing and Diagnostics

- Enabled best-effort indexing by default. Without `vide.toml`, or when a manifest omits `sources`, Vide now indexes Verilog/SystemVerilog files in the workspace so completion, go to definition, hover, reference search, and related editor features work out of the box. Semantic diagnostics still require explicit project configuration. Setting `sources = []` remains an opt-out for workspace indexing. ([#80](https://github.com/pascal-lab/vide/pull/80))
- Improved duplicate `module` handling in best-effort indexing. Explicitly configured source roots keep strict semantics and report duplicate definitions as ambiguous. Best-effort roots may resolve to the uniquely nearest candidate by path proximity, while tied candidates stay ambiguous and produce a `vide` diagnostic. ([#86](https://github.com/pascal-lab/vide/pull/86))

- Added diagnostic rule quick actions in VS Code. Diagnostics can now write rule configuration into user or workspace `vide.diagnostics.slang.rules`, making it easier to ignore or downgrade selected slang diagnostics. ([#88](https://github.com/pascal-lab/vide/pull/88))
- Simplified diagnostic and quick action wording, including duplicate-module diagnostics, so messages no longer expose internal indexing details. ([#90](https://github.com/pascal-lab/vide/pull/90))

## VS Code Experience

- Added LSP project status notifications and a single right-side Vide status entry in VS Code. The status menu now brings project configuration state, reload, manifest creation, server restart, and output access into one place. ([#90](https://github.com/pascal-lab/vide/pull/90))

![Vide status menu](https://github.com/user-attachments/assets/b3347995-aa80-4df8-a721-64be9819b738)

- Added status-menu creation for `vide.toml`. The generated manifest starts with empty-analysis defaults and commented example fields, making the transition from best-effort indexing to explicit project configuration easier. ([#90](https://github.com/pascal-lab/vide/pull/90))

![Create vide.toml from the status menu](https://github.com/user-attachments/assets/c2ca7277-a192-4a18-a223-8a78aae1a2f3)

- Moved `qihe` run logs into a dedicated `Vide Qihe` output channel. Command lines, stdout, stderr, and status logs no longer mix with the main language server output, and Qihe status is shown separately from the main project status. ([#92](https://github.com/pascal-lab/vide/pull/92))

![Vide Qihe output channel](https://github.com/user-attachments/assets/7e034dce-3888-4afb-9b65-08ae69660473)

- Added Chinese and English localization for user-facing VS Code strings, server diagnostics, code actions, code lens text, configuration errors, and Qihe messages. ([#87](https://github.com/pascal-lab/vide/pull/87))

## Fixes

- Fixed source map handling for AST nodes produced by include or macro expansion. HIR lowering can still consume those nodes, but source maps only store stable root-buffer locations, avoiding panics that could interrupt go to definition, reference search, and instance code lens. ([#93](https://github.com/pascal-lab/vide/pull/93))
- Fixed stale diagnostics after file deletion. Deleted files now return an empty diagnostic report, and workspace diagnostics are refreshed. ([#91](https://github.com/pascal-lab/vide/pull/91))
- Fixed end-structure inlay hint range filtering. Hints for closing structures such as `endmodule` are now anchored to the actual closing token instead of the full `ModuleDeclaration` range, so range-limited requests no longer return hints outside the requested range. ([#84](https://github.com/pascal-lab/vide/pull/84))
- Fixed project-state refresh after manifest creation, changes, and deletion. Ordinary source file loads no longer trigger unnecessary workspace reloads, and empty root-scan progress is suppressed. ([#90](https://github.com/pascal-lab/vide/pull/90))
