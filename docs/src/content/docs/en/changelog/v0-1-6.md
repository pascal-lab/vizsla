## Vide Branding and Project Configuration

- The project brand has moved from Vizsla to Vide across the Rust package, VS Code UI, docs, workflows, playground, and generated manifest schema. ([#161](https://github.com/pascal-lab/vide/pull/161))
- The VS Code extension install ID is now `pascal-lab.vide`, matching the current extension package name and publisher.
- The default project manifest is now `vide.toml`, and the old manifest compatibility path has been removed. New projects and documentation examples should use `vide.toml`. ([#161](https://github.com/pascal-lab/vide/pull/161))

## Editing Experience

- Port and parameter inlay hint labels are more compact. Same-name connections now show only the direction arrow, and parameter hints no longer include extra trailing space, reducing visual noise in instantiation lists. ([#147](https://github.com/pascal-lab/vide/pull/147))
- Code actions such as "Fill connections", "Fill parameters", and adding empty parentheses to implicit named ports can now appear without waiting for diagnostics. Module-instantiation snippets also remove `ports` / `params` placeholder text so inserted code is closer to the final shape. ([#153](https://github.com/pascal-lab/vide/pull/153))

## Runtime and Diagnostic Stability

- Request handling, shutdown, formatter runs, and Qihe runs now share clearer cancellation plumbing. Cancelled or stale Qihe runs no longer publish diagnostics, and external processes are cleaned up through cancellable process helpers. ([#136](https://github.com/pascal-lab/vide/pull/136))
- The background task pool now uses weighted fairness and contains panics at request and background-progress boundaries. Semantic-token delta cache updates are also applied as accepted main-loop response effects, reducing state drift when cancellation and reloads interleave. ([#136](https://github.com/pascal-lab/vide/pull/136))

## Playground and Online Experience

- Vide Playground is now maintained as first-class workspace code. The browser LSP, WASM entry point, and core analysis logic connect through a real crate API, and the docs site consumes the playground package directly instead of relying on external patches or committed generated assets. ([#160](https://github.com/pascal-lab/vide/pull/160))
- Multiple VideLab examples on the same page now respond only to their own virtual workspaces, avoiding duplicated inlay hints, definition navigation handled by the wrong client, and duplicate browser registrations for desktop-only commands. ([#169](https://github.com/pascal-lab/vide/pull/169), [#174](https://github.com/pascal-lab/vide/pull/174))
- The Chinese online editor experience now loads a language pack that matches the Monaco VS Code API runtime instead of the incompatible standalone `nls.messages.zh-cn.js` bundle. ([#176](https://github.com/pascal-lab/vide/pull/176))

## Documentation, Site, and Build

- The docs homepage was reorganized with feature showcases, comparison content, and clearer CTAs. Feature GIFs, examples, and screenshots now make navigation, completion, inlay hints, and missing-port repairs easier to preview. ([#163](https://github.com/pascal-lab/vide/pull/163), [#164](https://github.com/pascal-lab/vide/pull/164), [#166](https://github.com/pascal-lab/vide/pull/166))
- The user and advanced guides further separate daily-use features, project configuration, build/install notes, startup self-checks, and troubleshooting content. Chinese and English pages now match the current extension behavior and release matrix. ([#148](https://github.com/pascal-lab/vide/pull/148), [#165](https://github.com/pascal-lab/vide/pull/165), [#167](https://github.com/pascal-lab/vide/pull/167))
- Documentation preview and release workflows now handle OSS directory marker uploads, preview cleanup races, stale closed-PR uploads, draft 404 banners, preview comment permissions, and WASM build caching more reliably. ([#149](https://github.com/pascal-lab/vide/pull/149), [#151](https://github.com/pascal-lab/vide/pull/151), [#155](https://github.com/pascal-lab/vide/pull/155), [#157](https://github.com/pascal-lab/vide/pull/157), [#158](https://github.com/pascal-lab/vide/pull/158), [#159](https://github.com/pascal-lab/vide/pull/159))
- Released Rust language server `0.1.6` and VS Code extension `0.1.6`.
