## Qihe and Project Configuration

- Fixed Qihe startup on Windows when the working directory was passed as a `\\?\` verbatim path. Vizsla now passes ordinary Windows paths so Java and external toolchains can resolve the working directory correctly. ([#98](https://github.com/pascal-lab/vizsla/pull/98))
- Qihe analysis launched from VS Code editor title or file menu actions now uses the clicked file URI instead of the active editor. Command palette and keyboard shortcut invocations without a resource argument still fall back to the active editor. ([#109](https://github.com/pascal-lab/vizsla/pull/109))
- Added `vizsla.qihe.autoConfigureArgsFromManifest`, enabled by default to preserve existing behavior. When disabled, Qihe stops auto-adding manifest-derived arguments such as `--mode sv`, `--top`, `-I`, and `-D`; explicit `vizsla.qihe.compileArgs` and `vizsla.qihe.runArgs` continue to apply. ([#113](https://github.com/pascal-lab/vizsla/pull/113))

## Diagnostics Profiling

- Added the `Vizsla: Profile Diagnostics` command for current-file and workspace diagnostics. Profiling runs in a dedicated language-server session and writes trace summaries, folded stacks, static flamegraphs, and raw Chrome trace JSON. ([#110](https://github.com/pascal-lab/vizsla/pull/110))
- Moved the `Vizsla: Profile Diagnostics` entry into the unified Vizsla status menu instead of keeping it in the editor title bar. Users can still start current-file or workspace diagnostics profiling from the status menu. ([#118](https://github.com/pascal-lab/vizsla/pull/118))
- Interactive profiling now uses Speedscope assets bundled in the VSIX and opens in the external browser, keeping large trace rendering outside the VS Code webview process. ([#110](https://github.com/pascal-lab/vizsla/pull/110))
- Profiling sessions now pass only the configuration needed to start the dedicated profiling language server, avoiding temporary server configuration leakage into normal extension sessions. ([#114](https://github.com/pascal-lab/vizsla/pull/114))

## LSP Performance and Correctness

- Vendored slang source and fixed Windows/MSVC Release build optimization flags so slang C++ code is built with optimization. In the measured test scenario, default diagnostics improved from about 82s to about 16.6s. ([#102](https://github.com/pascal-lab/vizsla/pull/102), [#103](https://github.com/pascal-lab/vizsla/pull/103))
- Diagnostics now use profile-level slang compilation as the shared source for parse and semantic diagnostics, avoiding repeated parse/compile work for the same compilation roots and enabling slang's default warning group. In the same measured scenario, default diagnostics improved further to about 9.8s, while explicit `warnings=["everything"]` still reports the full warning set. ([#104](https://github.com/pascal-lab/vizsla/pull/104))
- Preprocessor index construction now avoids a full parse, reducing repeated work during diagnostics and indexing. ([#105](https://github.com/pascal-lab/vizsla/pull/105))
- Fixed named port and parameter connection resolution. Go to definition and semantic tokens now resolve against the instantiated target module instead of the local module scope. ([#107](https://github.com/pascal-lab/vizsla/pull/107))
- Fixed system task completion snippets by escaping the leading `$`, so accepting completions such as `$display` keeps the task name intact. ([#106](https://github.com/pascal-lab/vizsla/pull/106))

## VS Code Experience

- Improved Verilog/SystemVerilog TextMate grammar coverage. System task highlighting is now based on slang's known system task names, `.v`/`.vh` stay on the Verilog grammar, and `.sv`/`.svh`/`.svi` use the SystemVerilog grammar. ([#108](https://github.com/pascal-lab/vizsla/pull/108))
- VS Code-side error notifications for language-server startup, workspace reload, and related failures now offer a Show Output action. Qihe analysis failures offer Show Qihe Output and temporarily route the Qihe status entry to the matching log panel, making failure details easier to open. ([#117](https://github.com/pascal-lab/vizsla/pull/117))

## Internal and Maintenance

- Updated the VS Code extension development dependency `qs` to `6.15.2`, including upstream `parse` and `stringify` regression fixes. ([#112](https://github.com/pascal-lab/vizsla/pull/112))
- Fixed the Rust lint failure on `master` triggered by Clippy's `obfuscated_if_else` lint. ([#115](https://github.com/pascal-lab/vizsla/pull/115))
- Released VS Code extension `0.1.4`.
