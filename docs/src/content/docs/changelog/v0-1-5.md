## 诊断与工作区稳定性

- 诊断结果 ID 现在只会在真实诊断输入变化时更新，避免配置刷新但诊断内容未变时重复推送完整结果。（[#126](https://github.com/pascal-lab/vide/pull/126)）
- 文件加载、路径别名和工作区就绪状态的处理更稳定。Vide 现在会忽略过期的文件加载批次，统一真实路径与别名路径的文件身份，并在工作区准备好之前暂停诊断请求，减少打开工程或文件变化时的陈旧诊断。（[#127](https://github.com/pascal-lab/vide/pull/127), [#129](https://github.com/pascal-lab/vide/pull/129), [#131](https://github.com/pascal-lab/vide/pull/131)）
- workspace diagnostics 现在按诊断所有者组织，跨 source root、open URI alias 和 pull/publish diagnostics 场景的诊断投递更一致。（[#132](https://github.com/pascal-lab/vide/pull/132)）
- VS Code 启动时会把初始配置传给语言服务器，结构性配置变化也会触发工作区重新加载，减少配置修改后仍沿用旧项目模型的问题。（[#120](https://github.com/pascal-lab/vide/pull/120), [#121](https://github.com/pascal-lab/vide/pull/121)）
- 后台任务调度会优先处理延迟敏感的交互请求，降低大型工作区后台任务对编辑操作的影响。（[#125](https://github.com/pascal-lab/vide/pull/125)）

## 编辑体验

- 端口 inlay hints 现在可以指向对应端口定义，便于从实例化连接跳转到模块端口。（[#134](https://github.com/pascal-lab/vide/pull/134)）
- 命名端口语义 token 的范围更准确，避免只高亮连接表达式或遗漏端口名。（[#130](https://github.com/pascal-lab/vide/pull/130)）
- Verilog declaration list 的语法高亮更完整，改善 `.v` / `.vh` 文件中声明列表的颜色显示。（[#119](https://github.com/pascal-lab/vide/pull/119)）

## 项目配置、Qihe 与宏

- Qihe 项目分析现在会使用已解析的项目配置文件，`vide.toml` 项目会走工程分析路径。（[#143](https://github.com/pascal-lab/vide/pull/143)）
- manifest 中的预定义宏现在会进入共享的 IDE 解析和预处理状态，跳转定义、引用查找等普通 IDE 功能可以看到 manifest macro 配置。（[#142](https://github.com/pascal-lab/vide/pull/142)）
- 因预处理条件未激活的代码现在会作为 VS Code unused hint 淡化显示。该提示保留诊断元数据，但在编辑器中以 hint 呈现，避免额外的信息级蓝色波浪线。（[#144](https://github.com/pascal-lab/vide/pull/144), [#145](https://github.com/pascal-lab/vide/pull/145)）

## 文档、构建与发布

- 文档新增 Vide Playground，并更新用户指南，使安装、配置、命令、状态栏和 Qihe 说明与当前扩展行为保持一致。（[#135](https://github.com/pascal-lab/vide/pull/135), [#141](https://github.com/pascal-lab/vide/pull/141)）
- GitHub Pages 文档构建现在显式处理 base path，避免部署环境和本地构建的路径差异。（[#140](https://github.com/pascal-lab/vide/pull/140)）
- 本地 release metadata 默认值和语言服务器版本对齐逻辑已整理，减少 release 构建与扩展版本不一致的风险。（[#122](https://github.com/pascal-lab/vide/pull/122), [#123](https://github.com/pascal-lab/vide/pull/123)）
- 发布 Rust 语言服务器 `0.1.5` 与 VS Code 扩展 `0.1.5`。
