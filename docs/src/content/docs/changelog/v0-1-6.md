## Vide 品牌与项目配置

- 项目品牌已从 Vizsla 迁移到 Vide，Rust 包、VS Code 界面、文档、工作流、Playground 和生成的 manifest schema 已统一使用新的项目名称。（[#161](https://github.com/pascal-lab/vide/pull/161)）
- VS Code 扩展的安装 ID 已更新为 `pascal-lab.vide`，与当前扩展包名称和发布者保持一致。
- 默认项目 manifest 现在是 `vide.toml`，旧 manifest 兼容入口已移除。新项目和文档示例都应使用 `vide.toml`。（[#161](https://github.com/pascal-lab/vide/pull/161)）

## 编辑体验

- 端口和参数 inlay hints 的标签更紧凑。相同名称的连接只显示方向箭头，参数提示也去掉了多余尾随空格，减少实例化列表里的视觉噪音。（[#147](https://github.com/pascal-lab/vide/pull/147)）
- “补全连接”、“补全参数”和“为隐式命名端口补空括号”等 code actions 现在可以在没有诊断的场景下直接提供。模块实例化 snippets 也移除了占位文字里的 `ports` / `params`，插入后更接近最终代码。（[#153](https://github.com/pascal-lab/vide/pull/153)）

## 运行时与诊断稳定性

- 请求、shutdown、formatter 和 Qihe 运行现在共用更明确的取消链路。被取消或过期的 Qihe 运行不会再提交诊断，外部进程也会通过可取消的进程辅助逻辑清理子进程。（[#136](https://github.com/pascal-lab/vide/pull/136)）
- 后台任务池调整为带权公平调度，并在请求和后台进度边界隔离 panic；语义 token delta 缓存也改为在主循环接受响应后更新，减少取消和重载交错时的状态漂移。（[#136](https://github.com/pascal-lab/vide/pull/136)）

## Playground 与在线体验

- Vide Playground 现在作为仓库内的一等模块维护。浏览器端 LSP、WASM 入口和核心分析逻辑通过正式 crate API 连接，文档站点直接消费 Playground 包，不再依赖外部补丁或提交生成资产。（[#160](https://github.com/pascal-lab/vide/pull/160)）
- 同一页面上的多个 VideLab 示例现在只响应各自的虚拟工作区，避免 inlay hints 重复、跳转定义被错误客户端处理，或浏览器端重复注册桌面-only 命令。（[#169](https://github.com/pascal-lab/vide/pull/169), [#174](https://github.com/pascal-lab/vide/pull/174)）
- 中文在线编辑体验改用与 Monaco VS Code API 运行时匹配的语言包，避免加载不匹配的独立 `nls.messages.zh-cn.js`。（[#176](https://github.com/pascal-lab/vide/pull/176)）

## 文档、站点与构建

- 文档首页重新整理了功能展示、对比信息和入口 CTA，并补充了功能 GIF、示例和截图，让导航、补全、inlay hints、缺失端口修复等能力更容易预览。（[#163](https://github.com/pascal-lab/vide/pull/163), [#164](https://github.com/pascal-lab/vide/pull/164), [#166](https://github.com/pascal-lab/vide/pull/166)）
- 用户手册和进阶文档进一步拆分日常功能、项目配置、构建安装、启动自检和排障内容，中英文页面都同步了当前扩展行为与 release matrix。（[#148](https://github.com/pascal-lab/vide/pull/148), [#165](https://github.com/pascal-lab/vide/pull/165), [#167](https://github.com/pascal-lab/vide/pull/167)）
- 文档预览和发布流水线修复了 OSS 目录标记上传、预览清理竞态、关闭 PR 的过期预览上传、draft 404 横幅、预览评论权限以及 WASM 构建缓存等问题。（[#149](https://github.com/pascal-lab/vide/pull/149), [#151](https://github.com/pascal-lab/vide/pull/151), [#155](https://github.com/pascal-lab/vide/pull/155), [#157](https://github.com/pascal-lab/vide/pull/157), [#158](https://github.com/pascal-lab/vide/pull/158), [#159](https://github.com/pascal-lab/vide/pull/159)）
- 发布 Rust 语言服务器 `0.1.6` 与 VS Code 扩展 `0.1.6`。
