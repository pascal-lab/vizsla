## 破坏性更新

- **影响程度：高。** `sources` / `exclude` 现在按 workspace-relative shell glob 解释。原来的 `sources = ["rtl"]` 如果想递归包含目录，需要改成 `sources = ["rtl/**"]`；`exclude = ["build"]` 也需要改成 `exclude = ["build/**"]`。只影响已经配置这些字段且使用目录式写法的工程。（[#69](https://github.com/pascal-lab/vizsla/pull/69)）
- **影响程度：中。** 未配置项目配置文件、项目配置文件为空，或省略 `sources` 时，不再默认扫描整个 workspace 做语义分析，而是进入 syntax-only 模式。依赖自动全工程扫描的项目会失去语义诊断或跨文件分析，需要显式配置 `sources`、`include_dirs` 等字段。（[#47](https://github.com/pascal-lab/vizsla/pull/47)）

## 修复

- 修复部分工程中 `documentSymbol` / `definition` 触发 panic 的问题。根因是无路径/HIR 解析时错误展开了 `include`，导致语法节点范围不属于当前文件。（[#44](https://github.com/pascal-lab/vizsla/pull/44), [#46](https://github.com/pascal-lab/vizsla/pull/46)）
- 修复修改 `vizsla_config.toml` / `vizsla.toml` 后诊断不会及时刷新的问题，现在项目配置文件重载后会刷新 workspace diagnostics。（[#63](https://github.com/pascal-lab/vizsla/pull/63)）
- 修复 standalone `$` 后不会补全 `$display`、`$finish`、`$time` 等 system task/function 的问题。（[#61](https://github.com/pascal-lab/vizsla/pull/61)）
- 修复 `vizsla.toml` 被误当作 SystemVerilog 文件送去 slang diagnostic 的问题；现在 `vizsla.toml` 和 `vizsla_config.toml` 都会被识别为项目配置文件。（[#71](https://github.com/pascal-lab/vizsla/pull/71)）
- 修复 LSP trace 设置相关问题，支持 `$/setTrace`，并减少无意义的 trace 日志输出。（[#51](https://github.com/pascal-lab/vizsla/pull/51)）

## 改进

- `sources` / `exclude` 支持 glob pattern，例如 `*.v`、`src/**/*.sv`、`*_bb.v`。（[#69](https://github.com/pascal-lab/vizsla/pull/69)）
- 新增 `vizsla.toml` 作为推荐项目配置文件名，同时保留 `vizsla_config.toml` 作为兼容 fallback。（[#60](https://github.com/pascal-lab/vizsla/pull/60)）
- VS Code 插件新增自动创建工作区配置的能力：在检测到 RTL workspace 且缺少配置文件时，可提示用户创建默认 `vizsla.toml`。（[#47](https://github.com/pascal-lab/vizsla/pull/47), [#60](https://github.com/pascal-lab/vizsla/pull/60), [#66](https://github.com/pascal-lab/vizsla/pull/66)）
- 新增工程配置文件的 JSON Schema 产物和生成流程；默认配置会写入 `#:schema` directive。配置文件的编辑辅助主要交给支持 schema directive 的 TOML 工具处理，Vizsla 本身只负责读取配置并刷新工程模型。（[#59](https://github.com/pascal-lab/vizsla/pull/59)）
- `qihe` 集成支持按 workspace / 项目配置文件做项目级分析，不再只偏向单文件分析。（[#45](https://github.com/pascal-lab/vizsla/pull/45)）
- hover 信息增强：模块、端口、参数、task/function 等会显示更完整的签名，多来源定义之间也会更清楚地分段。（[#72](https://github.com/pascal-lab/vizsla/pull/72)）
- 新增参数化 module snippet，可补全生成带 `#(...)` 的 module 模板。（[#62](https://github.com/pascal-lab/vizsla/pull/62)）
- VS Code 增加 semantic token 样式支持。（[#65](https://github.com/pascal-lab/vizsla/pull/65)）

## 内部与文档

- 重构 code action 模块结构，拆分 action、diagnostics、collector、handlers、tests 等文件，主要提升维护性。（[#22](https://github.com/pascal-lab/vizsla/pull/22)）
- 文档迁移到 Astro Starlight，并更新配置相关说明。（[#52](https://github.com/pascal-lab/vizsla/pull/52), [#70](https://github.com/pascal-lab/vizsla/pull/70)）
- CI 避免 PR 分支重复跑同一套检查。（[#67](https://github.com/pascal-lab/vizsla/pull/67)）
- 发布 VS Code extension `0.1.2`。（[#74](https://github.com/pascal-lab/vizsla/pull/74)）
