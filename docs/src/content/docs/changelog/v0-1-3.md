## 索引与诊断

- 默认启用 best-effort indexing。没有 `vizsla.toml`，或 manifest 未配置 `sources` 时，Vizsla 会索引 workspace 中的 Verilog/SystemVerilog 文件，让补全、跳转、hover、引用查找等编辑能力开箱可用；语义诊断仍然只在显式项目配置下运行。显式配置 `sources = []` 可作为 opt-out，关闭 workspace 索引。（[#80](https://github.com/pascal-lab/vizsla/pull/80)）
- best-effort indexing 现在会更稳妥地处理同名 `module`。显式配置的 source root 仍保持严格语义，同名定义会被视为歧义；best-effort root 会在唯一最近候选存在时按路径邻近性解析，候选距离打平时继续保留歧义并给出 `vizsla` 诊断。（[#86](https://github.com/pascal-lab/vizsla/pull/86)）

![同名 module 解析改进后](https://github.com/user-attachments/assets/acdd45ee-444b-43a5-af7c-826fc6a8d5d4)

- 新增诊断规则快捷操作，可直接从 VS Code Diagnostics 将规则写入用户或 workspace 的 `vizsla.diagnostics.slang.rules`，用于忽略或降级特定 slang 诊断。（[#88](https://github.com/pascal-lab/vizsla/pull/88)）
- 诊断与快捷操作文案进一步简化，重名模块等提示不再暴露内部索引实现细节。（[#90](https://github.com/pascal-lab/vizsla/pull/90)）

## VS Code 体验

- 新增 LSP project status 通知，并在 VS Code 右侧状态栏提供统一的 Vizsla 状态入口。状态菜单集中展示项目配置状态、reload、创建 manifest、重启 server、打开输出面板等常用操作。（[#90](https://github.com/pascal-lab/vizsla/pull/90)）

![Vizsla 状态菜单](https://github.com/user-attachments/assets/b3347995-aa80-4df8-a721-64be9819b738)

- 可从状态菜单直接创建 `vizsla.toml`。默认生成空分析配置，并带有注释示例字段，方便从 best-effort indexing 过渡到显式项目配置。（[#90](https://github.com/pascal-lab/vizsla/pull/90)）

![从状态菜单创建 vizsla.toml](https://github.com/user-attachments/assets/c2ca7277-a192-4a18-a223-8a78aae1a2f3)

- `qihe` 运行日志迁移到独立的 `Vizsla Qihe` 输出通道，命令行、stdout、stderr 和状态日志不再混入主语言服务器输出；Qihe 状态也与主项目状态分离展示。（[#92](https://github.com/pascal-lab/vizsla/pull/92)）

![Vizsla Qihe 输出通道](https://github.com/user-attachments/assets/7e034dce-3888-4afb-9b65-08ae69660473)

- 新增中英文界面与诊断本地化，VS Code 面向用户的字符串、服务端诊断、code action、code lens、配置错误和 Qihe 消息会根据语言环境显示。（[#87](https://github.com/pascal-lab/vizsla/pull/87)）

## 修复

- 修复 include 或 macro 展开产生的 AST 节点 source map 处理问题。HIR lowering 仍可消费这些节点，但 source map 只保存 root buffer 中有稳定文本范围的位置，避免跳转、引用查找和 instance code lens 被相关 panic 打断。（[#93](https://github.com/pascal-lab/vizsla/pull/93)）
- 修复删除文件后相关诊断残留的问题；被删除文件现在会返回空诊断报告，workspace diagnostics 也会刷新。（[#91](https://github.com/pascal-lab/vizsla/pull/91)）
- 修复 end-structure inlay hint 的范围过滤。`endmodule` 等结束结构提示现在锚定到实际结束 token，而不是整个 `ModuleDeclaration` 范围，避免只查询模块体局部范围时返回范围外提示。（[#84](https://github.com/pascal-lab/vizsla/pull/84)）
- 修复 manifest 创建、变更和删除后的项目状态刷新；普通源文件加载不再触发不必要的 workspace reload，空 root scan progress 也会被抑制。（[#90](https://github.com/pascal-lab/vizsla/pull/90)）

## 内部与文档

- 文档新增英文站点，并补充解析与分析说明。（[#87](https://github.com/pascal-lab/vizsla/pull/87), [#80](https://github.com/pascal-lab/vizsla/pull/80)）
- 新增版本化 changelog 页面与发布前 changelog 检查。（[#75](https://github.com/pascal-lab/vizsla/pull/75)）
- VS Code 测试改用 l10n-dev 检查运行时字符串提取。（[#89](https://github.com/pascal-lab/vizsla/pull/89)）
- CI 在 Rust cache 失败时允许继续执行，减少外部缓存服务波动对检查的影响。（[#85](https://github.com/pascal-lab/vizsla/pull/85)）
- 发布 VS Code 扩展 `0.1.3`。
