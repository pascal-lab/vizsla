# 介绍

Vizsla 是面向 Verilog 和 SystemVerilog 的语言服务工具。我们把它拆成两个部分:

- `vizsla`: Rust 编写的 LSP 服务器, 负责读取工程、解析源文件、维护语义信息并响应编辑器请求。
- `editors/vscode`: VS Code 扩展, 负责贡献 Verilog/SystemVerilog 语言配置、语法高亮、命令、状态栏, 并启动语言服务器。

后续我们会支持更多编辑器。

VS Code 扩展当前识别这些文件类型:

| 语言 | 扩展名 |
| --- | --- |
| Verilog | `.v`, `.vh` |
| SystemVerilog | `.sv`, `.svh`, `.svi` |

服务器加载工程时还会读取 `.map` 文件, 以支持库映射等工程输入。

安装扩展后, 你可以在 VS Code 中获得这些 IDE 能力:

- 语法高亮和语言配置。
- 语法和语义诊断。
- 跳转到声明、跳转到定义、查找引用。
- 补全、悬停说明、签名帮助。
- 重命名、文档符号、大纲、折叠、选择范围。
- 全文格式化、范围格式化、按 Enter 时的格式化。
- 代码操作, 例如补齐端口/参数连接、把有序连接转换为命名连接。
- 语义高亮、inlay hints、模块实例 code lens。
